REPLY 2 (by nckw @ 2022-07-14T15:18:18.000Z):
Hi Luca,
Just to be sure, the command you pasted looks like the one needed to generate the distribution of the test-statistic that we use to calculate significances. This is only the first part since you also need to calculate the observed test-statistic (see http://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/commonstatsmethods/#observed-significance ).
For a -ve best fitted signal (which is what I assume happens since you get 0 for the asymptotic), you should also get 0 for the observed test statistic (by definition of the test statistic). Are you using the instructions from the link above to get -0.8?
Best,
Nick

REPLY 4 (by nckw @ 2022-07-14T15:52:11.000Z):
Hi Luca,
Ah ok, sorry, I misunderstood the -3.
The -T tells combine how many toys run inside the -M Significance command. Under the hood it is using a RooStats class and the -i tells combine how many instances of that class to make (so the total number of toys in the end will be i*T). I wouldn’t worry too much about the details there,
If you need lots of toys and find each run rather slow, its portable to run many jobs in the batch/grid and hence this is why the hadd command is there. If you don’t need multiple jobs however, just use the one output file as the input to the observed calculation.
Best,
Nick
