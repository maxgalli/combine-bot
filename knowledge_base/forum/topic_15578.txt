QUESTION (by afiqaize @ 2022-09-28T12:54:32.328Z):
Hi experts,
I am trying to understand the following remark [1] in [2]. However I struggle with the following bit:
“the nuisance parameter […] set to their nominal values which are obtained after fitting first to the data, with POIs fixed […]”
To what are the POIs fixed in this case? Their best fit values?
I’m asking mainly because this bit here:
“If you are using toysFrequentist, be aware that the values set by --setParameters will be ignored for the toy generation as the post-fit values will instead be used (except for any parameter which is also a parameter of interest).”
Running command [3] for two different POI values --setParameters (0, 0) and (2, 2), in both cases the “data” being sum of SM prediction prefit, and looking at the nuisance parameter values, I end up with very different nuisance parameter ranges [4] and [5]. Here I’m picking tmass only as an example, but almost all nuisances behave like this, particularly signal-sensitive ones.
Clearly there is a strong dependence on --setParameters, but if I’m understanding the manual correctly there should not be. What am I missing here?
Cheers,
Afiq
[1]
* `--toysFrequentist` the nuisance parameters in each toy are set to their nominal values which are obtained *after fitting first to the data*, with POIs fixed, before generating the data. For evaluating likelihoods, the constraint terms are instead randomised within their Gaussian constraint pdfs around the post-fit nuisance parameter values.

If you are using `toysFrequentist`, be aware that the values set by `--setParameters` will be *ignored* for the toy generation as the *post-fit* values will instead be used (except for any parameter which is also a parameter of interest). You can override this behaviour and choose the nominal values for toy generation for any parameter by adding the option `--bypassFrequentistFit` which will skip the initial fit to data or by loading a snapshot (see below).

REPLY 2 (by adewit @ 2022-09-29T08:18:03.488Z):
Hi Afiq,
The POIs will be fixed to the value you set it to by passing --setParameters  for the POIs, or the initial value in case they are not set.
This is what is meant by the comment in parentheses in the text you quoted: “If you are using toysFrequentist , be aware that the values set by --setParameters will be ignored for the toy generation as the post-fit values will instead be used (except for any parameter which is also a parameter of interest).” - ie values set by setParameters are ignored when you use toysFrequentist, except values set for POIs.
Hope this helps,
Adinda
