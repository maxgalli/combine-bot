Hi Andrew,
I made some small changes that allow running the impacts without
running the initial fit for the POIs, which can save a lot of time in
case the crossing points for the POIs are already known from running
scans for example [1,2]. For adding the crossing points of the POIs to
the output json file, another script is needed that is not included in
this commit, since it relies on scipy functionality only available in
python3.
In any case, these changes would allow people with complicated
likelihoods to extract the crossing points of their POIs with a custom
method and use the impact algorithm from combine harvester only for
running the fits for the impacts, which can save a lot of time.
Please let me know, if I should make a PR with this.
