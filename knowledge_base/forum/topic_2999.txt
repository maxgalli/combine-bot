QUESTION (by xuyan @ 2021-12-03T12:56:07.637Z):
Dear experts,
I am writing to see if it is possible to run the combine tool using a hybrid approach, i.e., use template signal histograms and parametric background shape? If not, do you have some suggested possible workaround? We expect difficulties in the signal modeling so it would be nice if we can stick to the signal template.
Thanks a lot for your kind help in advance!
Best regards,
Sebastian

REPLY 2 (by agilbert @ 2021-12-03T15:15:59.619Z):
Hi Sebastian,
Yes this is possible - the easiest way is probably to put your binned templates as RooDataHist objects with the same observable RooRealVar as the analytic pdfs - text2workspace.py will then convert these to an appropriate pdf.

REPLY 4 (by amarini @ 2022-01-13T18:07:16.000Z):
Hi,
There is a caveat to pay attention to the shape normalizations, if you use the ‘shape’ morphing in combine.
When you use ‘shape’ uncertainties in combine, and you pass them through TH1 or RooDataHist normalization differences between nominal and +/- 1sigma values are accounted for as a logNormal uncertainty correlated with the shape modifications.
When you convert RooDataHist into RooHistPdf, normalization information is lost, and therefore you will need to take care by hand for the normalization difference part, if present.
-Andrea
