REPLY 2 (by agilbert @ 2023-11-14T09:41:45.121Z):
Hi Cheng-Han,
I think your intuition is largely right - in principle adding categories should at best only improve the expected sensitivity. However, by default combine calculates the expected limit using a fit to the data, and it could be that your extra categories, via correlations with the signal region categories, increase the background normalisation there such that the sensitivity in the signal region is reduced.
To test this, you can run limits without fit to data first, see the instructions here: https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/commonstatsmethods/#blind-limits
cheers,
Andrew

REPLY 6 (by agilbert @ 2023-11-15T13:53:10.350Z):
Hi Cheng-Han,
My best guess: it seems some (all?) of your signal pdfs have a common set of dependent variable names, however in each input workspace these have different values. When you build the combined workspace, these parameters are merged - e.g. there is only one frac_dcb parameter - connected to all the signal pdfs, and taking essentially a random value (probably the values in the last pdf parsed by text2workspace).
I think you probably want to name the RooRealVars so they are unique for each pdf?
cheers,
Andrew
