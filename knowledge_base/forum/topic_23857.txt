REPLY 2 (by agilbert @ 2023-05-10T08:00:32.370Z):
Hi Sima,
So I guess what you want is to calculate the 1D confidence interval on each parameter of interest while the other two (and the nuisance parameters) are profiled?
We have a few different ways to do this, but the MultiDimFit method is a good place to start, there’s some documentation here:
https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/commonstatsmethods/#likelihood-fits-and-scans
The --algo singles option should run the scans for each parameter in turn and report the three intervals.
If you want to be able to plot the explicit likelihood scan then --algo grid is what you want - but you’ll need to run combine three times, one for each parameter, using -P [param] to identify the one you want to do the scan for.
cheers,
Andrew

REPLY 7 (by kcormier @ 2023-05-15T15:49:30.397Z):
Hi Sima,
I think the reason the plot is not working is just an issue with the parameter ranges; I think they may be too wide for what you are trying to do, so that a grid of 110 points is not capturing what you are looking for. There is also an issue that if you are looking for intervals on individual parameters while profiling the others, you should use the -P flag to specify which parameter you are scanning and plot the results of the 1D parameter scan. Otherwise you are not profiling the others, and while I haven’t checked the behaviour of plot1Dscan.py  in the case where the fit is done over multiple parameters, it won’t be what you want.
The reason the plotting script is complaining that there is only one point is because it cuts off points the deltaNLL > threshold. (which is configurable by --y-cut <value> on the command line, not that this is separate from the maximum value of the axis, which can be set with --y-max <value>).
Normally the y-cut value is 7, but taking a quick look at the limit tree in the file you provided the DNN values away from the best fit point are all in the hundreds. It seems at the granularity of your grid with those parameter ranges you are probing very unlikely regions of the parameter space.
I think the solution is to first follow Andrew’s suggestion of running MultiDimFit with --algo singles to see how the fit performs and what the sort of ranges of the uncertainty on each parameter are.
When using the --algo grid algorithm to scan a single POI, while profiling the others make sure to set which parameter you are scanning using -P.  Though its not necessary, you can use the result from the --algo singles run to get a better estimate of what range of the parameter to use.
Best,
Kyle
