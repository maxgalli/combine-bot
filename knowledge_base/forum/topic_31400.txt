REPLY 7 (by kcormier @ 2023-11-10T15:42:35.071Z):
Dear Afiq,
If you are not seeing any entries in the covariance matrix under shapes_fit_b/<bin_name>/total_covar it is possible that this is due to issues with the covariance matrix estimation.  See the online documentation on this [1]. If the covariance matrix is forced to be positive definite it will not fill the covariance matrix, as these values are really not to be trusted. You can check the status of the covariance matrix by printing the corresponding RooFitResult in FitDiagnostics output file.
Cheers,
Kyle
[1] https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/nonstandard/#fit-parameter-uncertainties
