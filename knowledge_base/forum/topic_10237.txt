Hello experts,
We are in the process of getting a CADI line and hopefully presenting AN-20-098 at ICHEP this year, in order to do that we request your feedback on our combine workflow which is documented here [1] (Those are the 20 bin data cards I described)
We had some trouble getting our run 2 limit to converge, it seemed to be that the limit derived from a histogram with 20 bins would converge while one with less bins, e.g. 15 would not converge, when we were using the command:
combine --run blind -M AsymptoticLimits -d  ws_vecu_emul_emul_run2_10bin_TTandST.root  -n setrto1-run2 --expectSignal 1
then we switched to :
combine --run blind -M AsymptoticLimits -d  ws_vecu_emul_emul_run2_10bin_TTandST.root  -n setrto1-run2 --minosAlg new
as this command allowed us to have limits that converged regardless of binning scheme.
Now however we still see this warning:
AsymptoticLimits.cc: 423 – Best fit of asimov dataset is at r = 0.000100 (0.010000 times rMax), while it should be at zero
This means that when fitting a background-only Asimov dataset, a non-zero signal strength is found, which should not be the case. The combine documentation says that this warning should not be ignored and that you should make sure everything is consistent in your setup.
So could you let us know what to do to try to resolve this non-zero signal strength issue?

REPLY 7 (by agilbert @ 2022-05-10T13:21:21.237Z):
Ok, so I ran
combine --run blind -M AsymptoticLimits -d 15bin/ws_vecu_emul_emul_run2_10bin_TTandST.root -n test
and indeed see that it only gets the median expected, not all the quantiles. Actually it looks like the cause of this is related to the default --rAbsAcc value, which is the absolute threshold on the accuracy to terminate the search for the limit. Since the default (0.0005) is of similar order to the 2.5% expected quantile, it seems the search truncates too early (though I appreciate combine is not at all informative on this). Should be fixable by running with --rAbsAcc 0 to only use the relative accuracy criterion, not the absolute. Can you confirm if this fixes things?
cheers,
Andrew
