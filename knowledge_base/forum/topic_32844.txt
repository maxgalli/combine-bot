REPLY 2 (by kcormier @ 2023-12-19T09:39:01.855Z):
Dear Jihun,
What you are seeing is most likely an artifact of the fact that your limit is small in units of r. Both HybridNew and AsymptoticLimits approximate the exact calculation of the limit in an iterative procedure and stop after some desired accuracy is reached.
The desired accuracy that you run with can be controlled via the command line options --rAbsAcc and --rRelAcc, which set the accuracy on r at which the algorithm stops.
The asymptotic method uses and analytic approximation of the test statistic distribution, which saves a lot of computational time, and therefore the default value of these parameters is fairly small (0.005 relative accuracy and 0.0005 absolute accuracy). On the other hand, the HybridNew method estimates the test statistic distribution using toy experiments, which is much more computationally expensive, therefore the default values of these parameters are much larger to avoid spending a lot of extra computation time on precision that may not be helpful, the defaults are 0.05 relative accuracy and 0.1 absolute accuracy.
So typically, if your limit is below 0.078125,  the algorithm will start at higher values, work its way down, get to this number and then stop because it is below the required absolute accuracy of the limit.
You can adjust this behaviour with the command line options above. Sometimes it is also useful to simply scale r such that the limit is ~1.
I hope that helps,
Kyle
