Hi Ram,
Yes, that’s the way to do it - just run more jobs on the set of signal strength points you want to increase the statistics for, using different random number seeds each time, then hadd all the results together. Combine will merge test statistic distributions coming from the same r value.

REPLY 12 (by agilbert @ 2022-03-06T15:20:41.702Z):
Hi Ram,
I think you changed the wrong option - IIRC crab is not like condor at CERN - the nCPU and memory are not tied together, so you need to set the value of maxMemoryMB in the config, not numCores. I would also consider running fewer toys in a single job, since there will be some correlation with the memory required.
cheers,
Andrew

REPLY 13 (by rdewanje @ 2022-03-06T15:36:28.292Z):
Hi Andrew,
The CRAB Twiki clearly says that:
maxMemoryMB integer Maximum amount of memory (in MB) a job is allowed to use. Defaults to 2000.
|numCores|integer|Number of requested cores per job. Defaults to 1. If you increase this value to run multi-threaded cmsRun, you may need to increase maxMemoryMB as well.

In the CMSSW parameter-set configuration you may require also the number of streams to be larger than one per thread, which affects the memory consumption too.

My colleagues here at KBFI told me that you can’t increase maxMemoryMB to more than 2000 or 2500 in a job since that is the maximum memory available per core in CRAB. So, if you want to increase your memory then the only way left is to increase numCores in the crab_Config.py (and numberofThreads per job in the CRAB PSet accordingly).
Correct me if I am wrong.
