REPLY 3 (by gortona @ 2025-10-30T14:08:05.000Z):
Hi Sasha,
I’m sorry but it is not 100% clear to me what you mean with your question.
When using discrete profiling, you are introducing a discrete variable in your likelihood function. For each value of the POI that you are testing, you search the minima across all the parameters space, including the discrete one and compute the likelihood ratio with respect to the global minima across all parameters. Is that what you were asking?
In practice, for each value of your POI, the discrete profiling will select the input function that maximises the likelihood function, like in the plot in the link you sent. The chosen function can be different at different value of the POI if that was your question.
I hope it helps,
Best,

REPLY 5 (by gortona @ 2025-10-30T15:12:24.000Z):
Hi Sasha,
Tthe plot shows the likelihood distribution without offset for the best fit. It provides no hint whatsoever about the actual input shapes used (besides the legend).
You can obtain the significance like you would do from a standard likelihood plot, simply subtracting the value of the likelihood ratio at the best fit point. In this case, extracting the values by hand, you have the best fit for r=0.5 at -2DeltaNLL~8512
For r=0 you have something like q0 = 2DeltaNLL(r=0) - 2*DeltaNLL(r=0.5) ~ 0.25 and he significance should be sqrt(q0)=sqrt(0.25)= 0.5
Best,
Giacomo

REPLY 7 (by gortona @ 2025-10-30T16:25:43.000Z):
Hi Sasha,
Yes the plot shows the likelihood distribution in absolute value with penalty term C. In principle, there’s no reason for likelihood obtain with different input functions to have the minima at the same r value, nor for them to have the same likelihood value at the minima.
When computing the significance you would of course subtract the value of the log-likelihood function at the best fit (8215 in this case) and use the black function (the envelope) as reference.
As you can see, by construction the black line will give you lower or equal significance than any of the individual functions (in our case is the same as the green function, but that’s by chance).
To be more explicit: in my understanding the value of the DNLL is taken from the lowest function at any given point of the POI
Best,
Giacomo

Sorry to jump in, but I think I’ve either misunderstood this statement, or its not correct. In either case I’d like to minimize any misunderstandings here.
Consider the case, for example, where one functional form (A) is a poor best fit (high absolute NLL), but is relatively broad (the NLL changes slowly with the POI), and another form (B) has a good best fit (low absolute NLL) but is very peaked (NLL changes quickly with the POI). For the sake of argument, lets say they have thesame best fit value of the POI. The significance using discrete profiling will tend to be larger than that of (A) but smaller than that of (B).
Just as in non-discrete-profiling case – adding uncertainties usually decreases significance, but this is not guaranteed to be the case and it can happen that it increases the significance if the uncertainty allows a better overall fit to the data. E.g. adding uncertainty on a background rate may allow that background rate to be pulled down by data in a control region and actually increase the overall significance of a signal; adding a functional form which is actually a better overall fit to the data may have the same effect.

Hi Sasha,
Yes the plot shows the likelihood distribution in absolute value with penalty term C. In principle, there’s no reason for likelihood obtain with different input functions to have the minima at the same r value, nor for them to have the same likelihood value at the minima.
When computing the significance you would of course subtract the value of the log-likelihood function at the best fit (8215 in this case) and use the black function (the envelope) as reference.
As you can see, by construction the black line will give you lower or equal significance than any of the individual functions (in our case is the same as the green function, but that’s by chance).
To be more explicit: in my understanding the value of the DNLL is taken from the lowest function at any given point of the POI
