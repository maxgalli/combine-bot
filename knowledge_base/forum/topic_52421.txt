REPLY 2 (by pkeicher @ 2024-10-10T08:22:48.179Z):
Hi Reza,
let me try to understand your setup a little better. Do you use one single category in your fit, or do you have multiple categories and want to consider only a subset of them for your b-only fit? What are the exact commands you use?
In case you want to use all categories that are defined in your datacard, freezing r to zero with --setParameters r=0 --freezeParameters r should do the trick. Note that this will probably not work as intended if you use the FitDiagnostics method, since here you perform two fits to start with - a signal+background fit (stored in the fit_s RooFitResult object) and a b-only fit where r is indeed frozen to 0 (which is stored in the fit_b object). If you just need a b-only fit, using the MultiDimFit method with the options above should work.
How do you create the pull plot? If you follow these instructions, remember to use the --skipFitS option to ignore any s+b fit.
To answer your other question: there isn’t a way to freeze parameters for a single process only, it’s always all or nothing. But so long as your parameters aren’t correlated to other processes, setting the signal strength modifier to zero should essentially deactivate these nuisances (i.e. they don’t have any effect in the fit). If there are correlations, you might see some pulls, depending on the (pseudo) data you use in the fit. But the yield of your signal process should still be zero.
Hope this helps! Best,
Philip

REPLY 4 (by pkeicher @ 2024-10-10T09:34:34.594Z):
Hi Reza,
Ah, I see you don’t mean the pull plots, but impact plots! That explains why --skipFitS doesn’t do anything, it’s meant for the diffNuisances.py script (which produces pull plots).
You can’t freeze r in the impact routine because the impacts are all about fitting r for different parameter configurations. More specifically, you first perform a global fit including all parameters, then you set each nuisance parameter to it’s +/-1 sigma post-fit values and recalculate the signal strength at this point. While you can freeze r=0 in the first step, the subsequent steps will ignore this since the objective there is to really fit r.
If you’re only interested in the pulls of your nuisance parameters, I would recommend using the workflow in the instructions I’ve linked above. This will essentially produce a plot with the pad in the middle of the impact plot you sent earlier.
Let me know if I misunderstood what you really want to look at, sorry for any confusion!
Cheers,
Philip
