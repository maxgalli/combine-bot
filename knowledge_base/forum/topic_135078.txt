REPLY 2 (by gortona @ 2025-10-22T10:08:50.721Z):
Dear Beren,
I only had time for a very quick look on your setup so far, so my answer might be incomplete. Please feel free to iterate further if you need more details.
First of all, on the command on slide 5. I think that the second definition of r_pos, the one scaling TTBar_2 dileptonic, is not being picked up by the workspace. You can check this by looking in the workspace with w→function(“r_pos”)→Print(“v”). This is because there can not be 2 objects with the same name in the workspace. If you need to scale TTBar differently in the 2 bins you will need 2 different objects. It should be fine since in any case they will depend on the same inputs (r_neg and Ac), so the correlation is preserved.
About missing inputs, this can mean that the fit failed while calculating the impact for that parameter. Can you open the higgsCombine_paramFit_Test_lumi_13TeV_correlated.MultiDimFit.mH125.root file and see if you get all 3 entries? If not, indeed there was some problem running the fit. My suggestion would be to re-run it only for the 2 missing systematics. You have the command inside the log. Sometimes it fails just out of bad luck, but if not it is worth investigating what is going on. It might be that for that particular parameter you need to fine tune a bit the command.
About the validation checks, I can suggest having a look at the covariance matrix, and maybe one can make a test by scaling by a large factor just one channel, but I don’t think we have standard tests we run in these cases on top of the standard ones from each PAG, so I’m not sure what to suggest here.
I hope it helps,
Best,
Giacomo
