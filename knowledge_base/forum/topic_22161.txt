QUESTION (by matariq @ 2023-03-31T19:39:05.206Z):
Dear Combine Experts,
I am trying to get postfit distributions for fit to Asimov dataset in Combine using PostfitShapesFromWorkspace.
I have my datacard named card.txt and use text2workspace.py to make a worksapce named card.root in the following way:

text2workspace.py card.txt

Then I run the fit using FitDiagonistics:

combine -M FitDiagnostics --saveWorkspace --saveToys --redefineSignalPOIs yt --freezeParameters r,EWunc,mtop --robustFit 1 -t -1 --do95=1 --cminPreScan --cminDefaultMinimizerStrategy 0 --cminFallbackAlgo Minuit2,Simplex,0:0.1 -m 125 -d card.root --setParameters r=1,EWunc=0,mtop=0

After that I get postfit distributions using PostFitShapesFromWorkspace:

PostFitShapesFromWorkspace -w higgsCombineTest.FitDiagnostics.mH125.123456.root -o shapes.root -m 125 -f fitDiagnostics.root:fit_s --postfit --sampling --freeze r=1,EWunc=0,mtop=0

In the above command, higgsCombineTest.FitDiagnostics.mH125.123456.root is the workspace which is saved in the 2nd command using saveWorkspace.
However, the 3rd command only saves post fit distributions for the real dataset (data_obs) instead of the Asimov dataset. I also save the Asimov data set using --saveToys as in the 2nd command but that saves it as a RooDataSet named model_sData in an entirely new workspace called toy_asimov . While using PostFitShapesFromWorkspace, I think by default only the workspace named w is imported which only contains the RooDataSet data_obs.
I know we can use the option --dataset to specify which dataset to use when running PostFitShapesFromWorkspace but since my asimov dataset dosen’t get saved in the same workspace as the data_obs, when I use the option --dataset model_sData with PostFitShapesFromWorkspace I get a Segmentation violation. How to get around this?

REPLY 4 (by agilbert @ 2023-04-04T12:52:39.088Z):
Hi Maida,
Right, at the moment there’s not a very intuitive way to do this.
A workaround I’ve used in the past is to first modify your nominal workspace and import the Asimov toy dataset into it. You can do this with combineTool.py, e.g.:
combineTool.py -M ModifyDataSet card.root:w card_with_asimov.root:w:toy_asimov -d higgsCombineTest.FitDiagnostics.mH125.123456.root:toys/toy_asimov
Then if you run PostFitShapesFromWorkspace on card_with_asimov.root you should be able to point it at toy_asimov.
Hope this solves the problem.
cheers,
Andrew
