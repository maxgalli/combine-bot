REPLY 2 (by nckw @ 2026-01-05T16:11:35.579Z):
Hi Georgios,
I think there may be some confusion about how the toys are generated. I will assume that for now we are only talking about the case without systematics (note there are different ways to account for these in the toy generation which you can find out about here: https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/latest/part3/runningthetool/#toy-data-generation)
Combine will use the pdf of the signal+background to generate toys. This means that for each toy, you cannot specify how many background or how many signal are generated, only the total number of events. Of course if you set the signal strength to zero, then you know all the toys are generated from a background only pdf. The number of events will be determined by the expected rate (of signal + background) which is defined by the datacard. Note that combine creates the “extended” term of the pdf so that the number is drawn from a Poisson with mean equal to the expected rate. So if in total you expect 100 events (i.e the rate in your datacard would be 100), then combine will generate N events in each toy, where N is drawn from a Poisson with lambda (mean) = 100.
Hope that clarifies things and please do read through the documentation on the toy data generation for details.
Nick
