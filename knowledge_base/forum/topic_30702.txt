REPLY 8 (by anigamov @ 2023-12-04T10:53:39.940Z):
Dear Sasha,
I think the easiest would be to add the following lines to the combined 2017+2018 datacard
ratio_2018_To_2017   param  1.0 <unc_size>
mu rateParam  <2017_region_name>  sig  1  [0.,5.]
r_2018 rateParam  <2018_region_name>  sig  @0*@1  ratio_2018_To_2017,mu

where is <unc_size> the uncertainty to assume for the efficiency ratios, <2017_region_name>  and <2018_region_name>  are the bin names for 2017 and 2018 categories.
Then for the fits (e.g.  -M MultiDimFit) you should add the --redefineSignalPOIs mu --freezeParameters r
Best,
Aliya

REPLY 14 (by anigamov @ 2023-12-08T15:04:07.710Z):
Dear Sasha,
I think the uncertainty is asymmetric because the ratio is restricted to be positive.
You could consider an alternative parametrisation, which would allow the nuisance parameter to vary in a wider range, but also make sure to have a positive definite ratio of signal strengths:  exp(ratio) = mu1/mu2, with ratio defined as param nuisance.
R2_R1  param  0  <unc_size>
mu1 rateParam  <region1_name>  sig  1  [0.,5.]
mu2 rateParam  <region2_name>  sig  exp(@0)*@1  R2_R1,mu1

Best,
Aliya

REPLY 16 (by anigamov @ 2023-12-13T14:21:38.140Z):
Dear Sasha,
I think you already have everything setup to run the combination. The floating parameters mu_sr1_2017, mu_sr2mod13_2017, mu_sr3_2017 are the new POIs and I think the significance can be estimated as discussed in the other thread with that Chernoff method for 3 parameters.
To combine the cards run the following:
combineCards.py SR3_2017+2018.txt SR2mod13_2017+2018.txt  SR1_2017+2018.txt > combined.txt

Running the fit:
combine -M MultiDimFit -d combined.txt --freezeParameters r --redefineSignalPOIs mu_sr1_2017,mu_sr2mod13_2017,mu_sr3_2017

It looks like some of the signal parameters are still the same in 2017 and 2018 workspaces, I think it might be a good idea to decorrelate them.
Best,
Aliya
