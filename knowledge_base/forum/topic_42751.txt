QUESTION (by anikiten @ 2024-06-25T11:03:57.996Z):
Dear All,
as you know we have 3 signal mu’s  and run feldman-cousins to evaluate a local significance with toys as
combine -d combined.txt -M HybridNew --LHCmode LHC-feldman-cousins --clsAcc 0 --singlePoint mu_sr1_2017=0,mu_sr2_2017=0,mu_sr3_2017=0 --saveToys --saveHybridResult -T 100000 -s -1 --setParameters mean=27.3,width=3.2 --freezeParameters mean,width,sigma --setParameterRanges mu_sr1_2017=0.0,100.0:mu_sr2_2017=0.0,100.0:mu_sr3_2017=0.0,150.0 --redefineSignalPOI mu_sr1_2017,mu_sr2_2017,mu_sr3_2017 -n run_1
and we got a p-value 1.3e-6 (corresponds to a local significance of 4.7) which coincide with one from the analytical formulae.
Now we want to evaluate LEE effect due to the dimuon mass range 25-30 GeV and width range of 0-5.6 GeV.  So, I run it like
combine -d combined.txt -M HybridNew --LHCmode LHC-feldman-cousins --clsAcc 0 --singlePoint mu_sr1_2017=0,mu_sr2_2017=0,mu_sr3_2017=0 --saveToys --saveHybridResult -T 1000 -s -1 --setParameterRanges mass=25.0,30.0:width=0.0,5.6:mu_sr1_2017=0.0,100.0:mu_sr2_2017=0.0,100.0:mu_sr3_2017=0.0,150.0 --redefineSignalPOI mu_sr1_2017,mu_sr2_2017,mu_sr3_2017 -n run_1
I was surprised to get p-value after running 1K events (just for test)
Pmu = 0.032 +/- 0.00556561
I think it is totally wrong number which correspond to significance about of 2.0.
Do we something wrong with command above or we should use some other command, since feldman-cousins does not work for LEE.
Thanks, Sasha.

REPLY 2 (by anigamov @ 2024-06-25T11:53:54.024Z):
Dear Sasha,
As far as I can see there are two differences between the commands you provided.
The first one uses more toys and also freezes the signal parameters, while in the second one the signal parameters are left freely floating.
So the way I would interpret these results is that the first output corresponds to the local significance and the second one to global, and there is nothing wrong about it. I think one could try to run more toys for the global significance (where the signal parameters are not frozen), but considering the uncertainty on p-value it would not change the results significantly.

REPLY 6 (by nckw @ 2024-06-25T13:42:52.000Z):
Hi Sasha,
In your model there is only one mass and width so combine cannot vary them independently across the SRs and years.
One (maybe minor thing) I noticed is that in your second command, should you not also have --freezeParameters sigma still? I don’t think that parameter should be free to float in either case.
I’m not sure I would trust the result straight away until we are sure that the free floating fits (ie for the global significance) are converging. In this case, you are generating events with no signal so I suspect the likelihood for many of these toys doesn’t have a nice easy minimum to find. For example, when we calculated the LEE for the Higgs discovery, we didn’t let the mass float, instead we performed a scan over the mass for each toy to find the deepest minimum in each case (same as saying the mass with the largest excess in each case),
I think it might be useful to try to run a 2D likelihood scan for a few of the toys you’re generating (just like you did with the real data) for the mass vs width in the ranges you are setting and see what it looks like. If its smooth then I think the command you have is ok, if its very “jumpy” we probably need a different approach. You should be able to run the same command for theis as you did for the data but just add the option -t 1 -s 1 to get a toy data instead of the observed data.
Best,
Nick

REPLY 10 (by nckw @ 2024-06-26T16:52:48.000Z):
Hi Sasha,
The point of looking at 1 toy is just to see what the likelihood surface looks likenfor a background toy. If there are lots of local minima, then when you try to calculate the p-value, many of the fits might be getting stuck in shallow minima and so the calculation is not reliable. If instead there are not lots of local minima, the calculation you performed already is probably correct,
Best
