QUESTION (by regartne @ 2024-01-26T10:49:05.854Z):
Dear combine experts,
I want to use combines HybridNew method to compute expected upper limits. I know that I need to specify the range for the background normalization sampling in the datacard. A colleague recommended using the 99% Clopper-Pearson interval as this range. However, combine raises an error whenever the lower bound of this range is bigger than 1. So, for example, [0.5,2] is accepted as a range, but [2,5] would raise a ValueError. Why is this the case?
When I use a function like https://www.statsmodels.org/devel/generated/statsmodels.stats.proportion.proportion_confint.html
to compute the Clopper Pearson interval, I believe I then have to scale it by the total number of tries and therefore usually get a range with a lower bound which is bigger than 1.
Am I missing something here or how can I resolve this issue?
Also, it would be helpful to mention somewhere in the documentation that there cannot be a blank space in the range, e.g. [0.5, 2] is not accepted by combine. It took me a while to figure that out.
Thanks in advance for you help.

REPLY 4 (by adewit @ 2024-01-26T16:23:15.489Z):
Dear Rebecca,
To me it looks to me like you get this error because the default value that you set for this parameter is 1, and the initial value cannot be outside of the specified range. I don’t think in general it makes sense to find an interval that does not include the nominal value; are you sure that you are calculating an interval around 1 and not around ~10?
Best wishes,
Adinda
