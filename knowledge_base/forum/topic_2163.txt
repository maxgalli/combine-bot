Hi,
I don't think that you can skip the combineCards or equivalent step (either as combineCards or combineHarvester),
as I doubt that the combineTool (for job submission) allows you to do it on the fly (maybe you can have it as a prepend command, but this will not change a lot ...).
The idea behind it is that the card should represent a complete input statistical model.
Although, (I don't see the full cmd line nor the dc, so you may already do it or may not be in the condition of doing it), if you are interested in a limit on other than 'r' parameters, do you know you can run the 'singlePoint' cls and check only the 'r=1' point to be (<>0.05); with great benefit on the timing for the limit computations, since you ran only two fits? (instead of eg. comparing the limit on r<>1)
Furthermore, quite a lot of time actually goes into the model building of the workspace creation (text2workspace), so it may be convenient to factorize out that step, especially if the model is large or if you want a convenient all-in file to ship somewhere (at the prize of fixing the physics model).
Cheers,
-Andrea

Hi,
the fast scan option, evaluates the likelihood without profiling it.
(so it plots L( v, \hat{theta}), rather than L(v,\hat{theta}_v) for any parameter v). This can be useful for if 1) minimizing the likelihood is consuming and you want a quick view of its value or 2) there is no other parameters (\vec{theta}==empty) and thus there is nothing to minimize.
You can check the value of the ABCD parameters, by looking at their postfit results (with or without r, accordingly to your preference). If you want to freeze r, you may define a new [set] of POIs, to make sure that r is not float (depending on the method used).
In MultiDimFit, if you are interested only in the central value of a parameter, you can use for instance:
--saveSpecifiedNuis arg               Save specified parameters (default =
none)
--saveSpecifiedFunc arg               Save specified function values (default
= none)
or the --saveWorkspace that will save the full workspace with a snapshot ('MultiDimFit) of the fit parameters at the best fit.
if you want to derive (profiled) likelihood scans on a parameter A, you can use:
-M MultiDimFit --redefineSignalPOIs A,B,C --algo grid --setParameterRanges A=... [--points 50] [--freezeParameters r=0] -P A --floatOtherPOIs=0/1 [--saveInactivePOIs=1]
On your original question, I forgot to mention -- at least if you use bash -- that you could in principle do the combineCards step with process substitution (i.e. bash constructs <(), combine -M ... -d <(combineCards.py ... ) ... ),
but I discourage it as the final card represents the inputs to your statistical results and it is useful to have them to debug, to understand what is inside, and to preserve.
Cheers,
-Andrea
