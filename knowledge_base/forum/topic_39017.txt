REPLY 2 (by nckw @ 2024-04-12T16:00:39.481Z):
Hi Matej,
Its possible that you’re finding a case where combine determines that the limit is accurate enough without the fit function (see here : https://github.com/cms-analysis/HiggsAnalysis-CombinedLimit/blob/main/src/HybridNew.cc#L516 ) - if you set the option --rAbsAcc X to something smaller, you may find it invokes the fit but by default this is set to X=0.1 (also there is a relative accuracy of 5% and combine uses the larger of the two).
The --noUpdateGrid option forces combine not to try to produce more points / update the toys, which by default it will do (again based on the toys). I’ve noticed there is an issue here in that with the --expectedFromGrid option set, the test statistic for the observation is also not updated and so indeed, you’ll get the same results every time. For now, you should remove that option, but its something we need to have another look at in the tool,
Best,
Nick
