QUESTION (by jspah @ 2024-04-09T13:43:57.998Z):
Dear Combine experts,
I am trying to set up Combine under EL9 (an slc9 architecture). I perform the following steps based on the instructions available at https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/latest/#compilation-on-lxplus9 to perform the compilation in an slc7 environment using apptainer.
git clone git@github.com:cms-analysis/HiggsAnalysis-CombinedLimit.git HiggsAnalysis/CombinedLimit
cd HiggsAnalysis/CombinedLimit/
git checkout v9.2.0
source /cvmfs/cms.cern.ch/cmsset_default.sh
cmssw-el7
. env_standalone.sh
make -j 4
exit
source env_standalone.sh

The compilation succeeds without any warnings on a) lxplus9 and b) on an EL9 machine of my institute (called lxbatch01), where I also have access to cvmfs. I can use Combine on lxplus9 with this setup. However, it does not work on the lxbatch01 machine. Even executing root fails due to shared library issues:
root
Fatal in <TROOT::InitInterpreter>: cannot load library libtinfo.so.5: cannot open shared object file: No such file or directory
which root
/cvmfs/cms.cern.ch/slc7_amd64_gcc900/lcg/root/6.22.08-ljfedo/bin/root

I checked that the basic ROOT setup is the same for lxplus9 and lxbatch01 in clean shells: The ROOT version is 6.30/04 and which root shows /usr/bin/root in both cases. After sourcing the standalone environment, the same path as above is shown also for lxplus9 with a ROOT version of 6.22/09.
Do you have any advice on how to set up Combine on EL9 machines where the configuration might differ slightly from the lxplus9 configuration?

As another option, I tried to set up Combine with conda (https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/latest/#standalone-compilation-with-conda). This actually works: The compilation make CONDA=1 -j 8 finishes without errors on EL9 (both lxplus and my local lxbatch01 machine) and Combine works nicely. This is a really convenient option to install Combine for lightweight use cases, thanks a lot to the developers for providing it.
