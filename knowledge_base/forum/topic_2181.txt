REPLY 2 (by agilbert @ 2021-10-13T09:09:44.000Z):
Hi Prafulla,
The impacts routine is doing what it's supposed to do - the issue is the non-active background parameters have no effect on the NLL value. So combine is scanning in each parameter, performing the fit, but will never find the deltaNLL value it needs to calculate the impact. This is why you get X +/- 0 returned for those parameters.
I think you have a couple of options:
- If you want to understand how these other background parameters would affect the signal strength if their associated pdf was the selected one, you could repeat the impacts calculation fixing the background pdf to that choice, and suppressing the discrete profiling. You'd have to repeat this for each pdf.
- If you want a more global picture of how the envelope procedure affects the measurement, you could make a plot similar to the one here: https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/nonstandard/#discrete-profiling, where the NLL curves for each pdf choice are overlaid with the envelope
Hope this is helpful.
cheers,
Andrew

REPLY 5 (by amarini @ 2021-10-13T15:50:24.000Z):
*** Discussion title: Higgs Combination
Hi Stephanie,

So do I understand correctly that it is normal to have very low

systematics on the background shape?
In order to assess the total impact of a discrete nuisance, and if this serve as systematic uncertainties, the scan suggested by Andrew is a better way.
( -> this https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/nonstandard/#discrete-profiling )
At the moment, we cannot conclude from what you linked if this is low or not.
The background estimation have uncertainties related to the free parameter of the shapes and the discrete nuisances.
The free parameters of the active (best-fit) shape are evaluated in the sense that you expect, since the likelihood is regular around the minimum on those parameters. The parameters of the non active shape do not influence the minimum of the likelihood nor they change its value if you move them a bit.
For instance if I look at the impact plots you circulated,
you have two categories, leptonic and hadronic, with fairly low counts (norm) and therefore the associated normalization is high impactful (~sqrt N). The best-fit pdfs are one Bernstein and one Exponential. These have relatively low number of degrees of freedom (probably because you expect few events) and the normalization happens to be more important than the actual shape degrees of freedom. The non active shapes are exponential, landau, powerlaw, and so on ... their parameters don't change the likelihood function when you scan them directly, one by one, as they can at most worsen the minimum of a non best-fit pdf. Therefore the profiled likelihood will never switch as you change one of them alone. To understand them, you need a different definition (not the one in the impacts) that accounts for these discreteness and for the activation of the new parameters.
If you now want to conclude something about the other pdfs you should run the scans above, such that scanning the likelihood as function of r (and not as function of one of the inactive parameters, as in the impacts), you can see the discrete parameters moving around and changing their values. In particular, you can compare the scan with respect to the best-fit or an a-priori choice (I think you are still blind so this may be identical).
Further, notice that usually coverage properties are studies on toys or Asimovs since they may include realizations that didn't happened in the data observation we have (but I don't think you are doing the observed impacts).
if you want to derive how important are the free parameters in a non active pdf for that pdf choice, you should run the impacts as suggested by Andrew; but I guess this was not what you were asking for.
Cheers,
-Andrea

Hi Stephanie,
in general, there is to consider that there are some optimization to the fitting routine in order to keep the fitting time for discrete nuisances reasonably small (especially with large number of categories). I would not be particularly worried as long as the deltaNNL is small and many sigma separated from the best-fit.
In this case, you probably would not have a noticeable quotable difference on the two sigma interval (deltaNLL = +4).
If you want, you can try to test it (to my knowledge, but someone else can jump in if I say something inaccurate) by running again with --cminRunAllDiscreteCombinations, but it should be clear that it will become more and more prohibitive (as running time) for the more and more categories you will fit simultaneously. However, I don't see if this test will give you an answer you are seeking for.
Cheers,
-Andrea

Hi all,
After some discussion off the HN, the issue is now understood. In this specific case with no lnN/shape systematics in the datacard, text2workspace does not define the discrete parameter correctly in the output workspace, hence no discrete profiling with MultiDimFit.
Fix is rather simple: https://github.com/cms-analysis/HiggsAnalysis-CombinedLimit/pull/701
Prafulla - you will need to apply this manually until it makes it into the next combine release.

Andrew
