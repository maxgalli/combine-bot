REPLY 2 (by agilbert @ 2025-02-14T09:52:44.151Z):
Hi Attilio,
Some quick responses:

Possible that it’s not strictly necessary in the envelope scan, it may just be that you have to manually fix and offset between that scan and the scans done with each PDF fixed individually. Or put another way - you should be able to obtain the same final plot in both cases. If not, suggest to keep REMOVE_CONSTANT_ZERO_POINT=1.
Yes, that should be ok.
I think setting the binning will be enough, but again, it could be good to check.
In principle negative values shouldn’t happen, it implies that the real data agrees better with the model than a model perfectly matching the data. This could imply some problem in the setup (perhaps related to point 3). I would suggest to try and understand this first and not just remove these values.
Whether the discrete profiling impacts the significance or not for a particular data set depends on whether the best-fit function changes between the best-fit signal strength and mu=0, but over the ensemble, yes, the uncertainty in the choice of bkg function should be taken into account. There is no hard rule about the number of functions to include in the profiling method. The goal should be to consider a diverse set of functions that can adequately describe the data. If someone finds a fourth function that can describe the data but with a very different fitted signal strength, then there is no promise that the envelope method with 3 functions will have the correct coverage. But this is always the case whether we use the envelope method or some other procedure to assign an uncertainty for the choice of background function.

cheers,
Andrew
