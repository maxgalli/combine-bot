REPLY 2 (by agilbert @ 2026-03-13T15:03:23.308Z):
Dear Kuan,
There’s unfortunately no easy way to get autoMCStats and RooParametricHist working nicely together. This request has come up a few times in the past, but it would require writing essentially an entirely new version of CMSHistErrorPropagator that can take arbitrary binned RooFit pdfs as input (which carries quite a few complications).
Solutions are:

stick with a mixture of RooParametericHist and RooHistPdf, implement MC stat uncertainties manually as shape variations per bin and per (relevant) process
stay with TH1 input - instead of using RooParametricHist it may be possible to split your QCD template up into N separate datacard processes (where N = number of histogram bins). Then your can apply formulae for each bin yield via rateParams, and keep using autoMCStats.

Cheers,
Andrew
