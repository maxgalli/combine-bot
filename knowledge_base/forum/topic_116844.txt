REPLY 4 (by jlangfor @ 2025-02-27T17:04:16.248Z):
Hi Yihui,
We had the same problem when trying to add the offshell analysis regions into the Run 2 Higgs combination, as they have a large number of bins. By default, all channels will share the CMS_th1x variable unless explicitly stated, which grows in dimension with every channel added.
We applied this fix [1]. Analysis regions with bins over some threshold (default 1000 bins) will automatically get a new 1D variable. It’s more memory intensive to have a separate variable for every analysis region, so there’s some compromise. You can tune this threshold according to your setup.
We should probably think of a better fix moving forward which means variables for each region are treated optimally, but this worked for us.
Cheers,
Jon
[1] - https://github.com/cms-analysis/HiggsAnalysis-CombinedLimit/commit/1b6dd8bfb69be7fc8e54305aa8c2384bd853af8c
