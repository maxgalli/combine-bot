QUESTION (by mbonanom @ 2023-07-27T09:27:35.990Z):
Dear all,
I am writing here to ask for a feedback about a technical feature we just spotted while running FitDiagnostics.
We are combining a set of cards and when doing so with combineCards.py we specify the name of the region that each card corresponds to. Our region names include some dashes (eg SR-6j-2b), so our command looks like this:
combineCards.py SR-6j-2b_diMuon_ULRunII=ULRunII/AZH_MA-650_MH-450_2DEllipses_diMuon_SR-6j-2b.dat .... > RunII_Card.dat

when running FitDiagnostics  we get the following error:
[#0] ERROR:Eval -- RooFormula::eval(CMS_channel==CMS_channel::SR-6j-2b_diMuon_ULRunII): Formula doesn't compile: CMS_channel==CMS_channel::SR-6j-2b_diMuon_ULRunII

which gets triggered by the fact that we have - in the channel names.
This looks like more of a RooFit than a combine issue, as the name of the channels is not parsed correctly by RooFormula::eval.
If we remove the - from the region names when combining the cards, the error above vanishes. The fit results in the two cases (ie using a combined card where the channels have a - in their name and when they don’t) are identical. However, I was wondering whether this is a known feature and it can cause troubles anywhere else in the fit (or elsewhere, in the usage of CombineHarvester for example). Is there anything that discourages the usage of - in the definition of the regions/channels ?
Thank you very much in advance for any feedback!
Cheers,
Matteo

REPLY 2 (by agilbert @ 2023-07-27T14:30:12.186Z):
Hi Matteo,
Off the top of my head, that error is probably triggered by the part of the code that generates the output for saveShapes (*), in this case for the data_obs. Did you check if this is correct in both cases? I suspect in the version with the ‘-’ this will not look correct. It’s probably not a bug as such - I guess roofit has no way of knowing where the “-” is part of a string or a minus operator.
I don’t think there are many other parts of the code where we do this (and not in CombineHarvester). So you’re probably safe, with the caveat on the saveShapes output.
cheers,
Andrew
