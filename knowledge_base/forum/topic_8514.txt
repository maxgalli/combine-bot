QUESTION (by fuyan @ 2022-03-22T22:33:17.379Z):
Dear Experts,
I’m working on topEFT analysis with multiple Wilson coefficients (WCs) being the POIs. I’ve been using the MultiDimFit method with -t -1 option, which is supposed to fit to the Asimov dataset. The command I’m using is:
combine wps_lj0pt_nonuis.root -M MultiDimFit --saveFitResult --redefineSignalPOI cpt
The fit results[1] of the WCs are non-zero. However, the Asimov dataset corresponds to the SM case where all the WCs are zero. Thus fitting to the Asimov data would give all 0 for the postfit values of the WCs. Besides that, I expect the prefit values of the parameters should be the default values pulled from the workspace[2], which are all 0. But it’s not the case. I’m loading floatParsInit() as the prefit values and floatParsFinal() as the postfit values.
I’m wondering if there is something I’ve been missing when doing the fitting, or maybe I’m looking at the incorrect set of the prefit and postfit results?
Attached is the fit results[1], the workspace[2] and the datacard[3].

REPLY 7 (by agilbert @ 2022-03-25T08:20:39.907Z):
Hi,
I think there are a few things going on:

The floatParsInit values are not the real pre-fit values in this case. The analytic minimisation of autoMCStats parameters is enabled by default (i.e. they are frozen so as not to be seen by Minuit), but since you requested the RooFitResult, we run a 2nd fit after the initial fit with the autoMCStats parameters re-floated, so that they appear in the Hessian matrix. This means the initial values for this 2nd fit are the outcome of the 1st main fit. So there’s no problem here.
The problem of the non-zero fitted values is that the Asimov data set is being generated from the background-only model by default. So excluding whichever processes in the datacard you marked as signal. Combine should be warning you about this, I get the message: Warning! -- You haven't picked default values for the Parameters of Interest (either with --expectSignal or --setParameters) for generating toys. Combine will use the 'B-only' ModelConfig to generate, which may lead to undesired behaviour if not using the default Physics Model

When --setParameters is used, the fit does recover the correct pre-fit values, but indeed as Adinda said, it seems there are problems calculating an accurate covariance matrix. This suggests you have some strong degeneracy in the set of parameters you are trying to fit.

cheers,
Andrew
