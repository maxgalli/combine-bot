REPLY 2 (by nckw @ 2023-07-27T15:51:51.075Z):
Hi Brooks,
It might help to let us know a bit more about what it is you’re trying to achieve but based on the fact you say you want to “plot the nuisance parameters at specific values of r”, i’ll assume what you want to know is what the fitted nuisance parameter values are for a range of r values.
Now, the simplest way to do this in combine is to instead use the -M MultiDimFit method with the --algo scan option. This produces a TTree in the output which is a scan of the negative log-likelihood as a function of the fixed parameter of interest (in this case r). You can include the option --saveSpecifiedNuis=all to get combine to also store the values of the fitted constrained nuisance parameters (i.e not flatParams) in the output TTree. From this you could make the plot you want I think. If you also want to include flatParams, rateParams etc, you can add another option --trackParameters x,y,z... so that combine also includes them in the TTree with the name trackedParam_x,y,z....
I hope this is helpful and what you were looking for,
Best,
Nick

REPLY 4 (by nckw @ 2023-07-27T20:22:45.040Z):
Hi Brooks,
If you prefer to use FitDiagnostics, you could try using the options --customStartingPoint --setParameters r=specific_value. FitDiagnostics performs two fits, one with the signal strength floating and another with it fixed by default to 0. This option will override the default value of 0 in the latter fit and then the nuisance parameter values in your output “b-only” results will in fact be the result of the fit with r fixed to your specific value. Would this work for your setup?
Best,
Nick

It may be that for your particular value, the fit failed and that could mean the fit_b result is not produced.
There is one more way I can think you could try which is to redefine the list of parameters of interest to be one of your nuisance parameters that is rather negligible (in fact you could create one in the datacard that essentially does next to nothing for this purpose) with --redefineSignalPOI nuis. Now the two fits will be one in which that nuisance parameter is floating and one in which it is fixed to 0 but in both cases you can use --freezeParameters r --setParameters r=some_value to fix the (now no longer POI) r to whatever you want in both fits. Beyond this i’m out of ideas as its already not really the purpose of FitDiagnostics, and even with this suggestion, i’d say we’re in square-peg round-hole territory.
