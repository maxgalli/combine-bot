combine -M GoodnessOfFit -d 2017UL_dimuon_SR2mod13_BestFit.root --algo=saturated --toysNoSystematics -t 500
and get the 500 numbers looks like:
Best fit test statistic: 98.4113
Generate toy 500/500
Looking into code I have the impression that this is kind of Chi2 according saturated model but
it is not clear is it normalized to NDF.
For BG only case I use:
combine -M GoodnessOfFit -d 2017UL_dimuon_SR2mod13_BestFit.root --algo=saturated --toysNoSystematics -t 500 --freezeParameters r --setParameters r=0
and again get the same kind of numbers flowing over screen.
Both attempts produce some root file higgsCombineTest.GoodnessOfFit.mH120.123456.root with
tree “limit” with 500 of  quantileExpected=-1 and empty directory with toys.
Could you advice what I am doing wrong and how I could compare these two cases with GoodnessOfFit: S+B and B?

REPLY 2 (by byates @ 2024-01-19T13:35:01.000Z):
Hi Olga,
This isn’t an “official” response, since I’m not a part of StatComm, but I’ve done this exact thing a few times. You are correct that the saturated model is a generalization of the chi-square test [1]. What you’ve done with the -t 500 flag is generated a test statistic of 500 toy datasets, where combine starts with an Asimov dataset and fluctuates each bin in the workspace using the Poisson statistics. What you then have to do is run combine one more time without the -t flag to get a saturated test value for your data.
combine -M GoodnessOfFit -d 2017UL_dimuon_SR2mod13_BestFit.root --algo=saturated
Then, to get your p-value, you find the number of toys above the data value, and divide by the total number toys. You would do this for both the background only model (toys) and the signal + background model (toys) to see if the data favors one over the other (i.e., if the B only model gave a p-value below some threshold, this would indicate the data does not favor this model).
Hope this helps, and that StatComm is on board.
[1] https://www.physics.ucla.edu/~cousins/stats/cousins_saturated.pdf
