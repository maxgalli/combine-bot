REPLY 11 (by nckw @ 2024-04-25T10:31:35.966Z):
Hi Sasha,
Since you pointed me to your datacard, i’ve had a go myself and here is what I did

I copied over your files to my own local area
I run the same text2workspace command (I see now why adding the --redefineSignalPOI didn’t change things since you already used a physics model to define them, I didn’t realise that before, but in any case I left the command in since it doesn’t hurt) :
text2workspace.py -P HiggsAnalysis.CombinedLimit.PhysicsModel:multiSignalModel --PO verbose --PO 'map=SR1_2018/sig:r_sig1[1,0,120]' --PO 'map=SR2_2018/sig:r_sig2[1,0,120]' --PO 'map=SR3_2018/sig:r_sig3[1,0,120]' 2018UL_dimuon_SR1SR2SR3.txt -o 2018UL_dimuon_SR1SR2SR3.root
I created a .sh file with the combine line needed to generate the toys which looks like below,

#!/bin/bash
cd /afs/cern.ch/work/n/nckw/combine-versions/113x/CMSSW_11_3_4/src/HiggsAnalysis/CombinedLimit/forSasha/comb/jobs/
cmsenv

i=$1

combine ../2018UL_dimuon_SR1SR2SR3.root -M HybridNew --LHCmode LHC-feldman-cousins --clsAcc 0 --singlePoint r_sig1=0,r_sig2=0,r_sig3=0 --saveToys --saveHybridResult -T 500 -s -1 --setParameters mean=27.3,width=1.4 --freezeParameters mean,width,sigma --setParameterRanges r_sig1=0.0,120.0:r_sig2=0.0,120.0:r_sig3=0.0,120.0 --redefineSignalPOI r_sig1,r_sig2,r_sig3 -n run_${i} --cminDefaultMinimizerStrategy 0

Note that it also fixes the parameter sigma since the range was set to not allow it to float anyway (easier just to make it constant). I also used --cminDefaultMinimizerStrategy 0 since it makes the fits for the toys a bit quicker.
I then submitted about 50 of these jobs to get 25000 toys in total (takes about the time it took to make a coffee and read a few emails etc).
Then I collected the output from the jobs together with hadd into a single file which I called toys.root and I run the following command
combine 2018UL_dimuon_SR1SR2SR3.root -M HybridNew --LHCmode LHC-feldman-cousins --clsAcc 0 --singlePoint r_sig1=0,r_sig2=0,r_sig3=0 --readHybridResult -s -1 --setParameters mean=27.3,width=1.4 --freezeParameters mean,width,sigma --setParameterRanges r_sig1=0.0,120.0:r_sig2=0.0,120.0:r_sig3=0.0,120.0 --redefineSignalPOI r_sig1,r_sig2,r_sig3 -n result --cminDefaultMinimizerStrategy 0 --toysFile toys.root

and the output is

-- Hybrid New --
Pmu = 0.002 +/- 0.00028256
Done in 0.00 min (cpu), 0.01 min (real)

Finally, since I had the script available to do it, I also plotted the distribution of the toys and the formulae from Fowlie on top (see below).

As you can see, things look nice - in my calculation, I got 2DeltaNLL=12.72. Plugging into the Fowlie formulae for n=3, I get a p-value of 0.00144 so pretty close to what we get from the toys. I am running a few more toys now just to see if the agreement gets better, but to me given the agreement between the line and the histogram I would say we’re ok.
So, with the  toys calculation (p-value=0.002), one would say the significance is  2.88 sigma.
