QUESTION (by zhguo @ 2025-02-26T12:58:14.317Z):
Dear experts, I’m writing to report a problem encountered when using combine:
-I’m using el9, CMSSW_14_1_0_pre4 as my setup environment, I guess the combine version is 10(but not very sure)
The el9 combine cannot understand an expression with more than one variable with the same name. For example, in the br_scaling function:
self.make_expr(“expr::{}(‘@0 * @1 * @2 * @3’, {}, {}, {}, {})”.format( xsbr_scaling, xs_scaling, br_scalings[0], br_scalings[1], br_scalings[2],))
Where all the br_scalings are “CVktkl_BRscal_hbb”, the el9 combine will crash when running this function.
Also, the same problem occurs with constants, like in the function below:
Self.make_scaling(“ggH”, Cb=”1”, Ctop=”kt”, Cc=”1”)
With both Cb and Cc defined as “1”, the function fails, this may also happen when I call functions in SMHiggsBuilder in combine.
This problem does not exist in the el7 combine, which is used by inference, but I still preserved the changes to the model aiming to running with el9 combine when testing. In the meantime, it seems keeping things as is and using CMSSW_14_0_0_pre1 and sticking to combine -v9.1.0 seems to work for el9 as well
Commands used:
text2workspace.py kl_scan/ProbHHH6b_3bh0h_inclusive_kappa_mod_rename.txt -P HiggsAnalysis.CombinedLimit.hh_model_basisV3:model_hhh_ggf --mass=125 -o workspace_ggf_rename.root
combine -M MultiDimFit -t -1 --algo grid -d workspace_inference_d3h.root --mass=125 --setParameterRanges kl=-300,300 --redefineSignalPOIs kl --setParameters r=1,r_gghh=1,r_gghhh=1,kt=1,k4=1,kl=1,CV=1 --freezeParameters r,r_gghh,r_gghhh,k4,kt,CV -n combined_rename_asi.root --saveWorkspace --verbose 3 –saveNLL
The command with crash because of the problem mentioned above.
The crash is because the workspace from text2workspace is a broken one, while no error is reported in the text2workspace step.
The model and cards used are attached below:

REPLY 4 (by anigamov @ 2025-04-10T13:08:33.157Z):
Dear Zhengliang,
Thanks for posting the issue, I am answering here because I do not think that it is Combine issue. I I think you are not using the physics model syntax in the intended way. The reason it stopped working with the recent CMSSW version is due to the significant refactoring of RooFormulaVar class since ROOT 6.30. In this case, where you want to set \kappa_b to 1 and freeze it I would recommend creating a const \kappa_b variable self.modelBuilder.factory_("kb[1.0]")and then parsing it to the model, but I think you should avoid this syntax :

Zhengliang Guo:

self.make_scaling(“ggH”, Cb=”1”, Ctop=”kt”, Cc=”1”)

Best,
Aliya
