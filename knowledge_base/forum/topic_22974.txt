REPLY 4 (by adewit @ 2023-05-02T21:04:04.679Z):
Hi Seth,
Apologies for the slow response - the issue indeed seems to be related to the fact that you were calculating the post-fit expected significance, and given you have set the observation to 0 in the datacard, the significance is also more or less 0. To me it looks you then end up with -inf as a result of some numerical instabilities.
When you calculate the pre-fit expected significance, what you’re doing is almost right, but to generate the asimov dataset remember to add --expectSignal 1 (or some other nonzero signal strength). Otherwise the issue remains since the background expectation is so small that N in the asmov dataset is ~0.  In addition, you don’t need -s -1 for the asimov dataset generation (in this case the seed doesn’t do anything, so it also doesn’t do any harm, but the file containing the asimov dataset will have a friendlier name without it)
Best,
Adinda

REPLY 9 (by anigamov @ 2023-05-25T12:53:03.544Z):
Hi Seth,
I’m sorry for a delayed reply, quite a few of us looked at this issue, let me try to summarise what we’ve found so far.
The disagreement between the two values can be understood if we look at the test statistics distribution (attached to this post), you can see the distribution is very discrete and the majority (>50%) of entries are 0, hence the median test statistics value = 0, which results in the p-value for 0.5 quantile at 1. This plot also shows that the asimov “observed” significance does not correspond to the median expected significance from the toys. It might be possible that with a larger sample of toys one would be able to populate this distribution better and get a better agreement between the asimov and expected values, but this has to be tested.
Someone else might want to comment further. Let me just say that we’ll get back to you with better recommendations, after some studies with more toys. But for optimisation studies it should be fine to use Asimov or asymptotic values as long as you are consistent.
Best,
Aliya
