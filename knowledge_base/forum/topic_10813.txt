REPLY 2 (by agilbert @ 2022-06-03T10:03:48.754Z):
Hi Prafulla,
Sorry for the slow reply on this. I didn’t have a chance to look at this in detail yet, but there might be a couple of things you could try:

The argument to -T (number of toys per batch) is a little low, I’d suggest setting this to something O(1000), at least for debugging.
Possible the scan for the limit is terminating early because of the absolute accuracy requirement (this is often a problem if the absolute value of the limit is very small) - you could try setting --rAbsAcc 0

It’s probably not a good idea to set such a large --clsAcc value either - could you try leaving this as the default - or if the runtime is really too prohibitive then only increase by a small factor

If none of these address the problem I’ll try to look in more detail.
cheers,
Andrew
