QUESTION (by amassiro @ 2026-02-12T10:35:09.425Z):
Dear colleagues,
I was wondering if the following tool already exists: a multi-analysis pull/impact.
Let me contextualise: in view of combination efforts, but also analysis spanning over several years of data taking, given the json files of the different pulls and impacts, it would be of great usefulness to have a visual comparison of the common nuisances, to check pulls and constraints of the different groups of datacard.
The goal would be to have:

rows: nuisances
columns: pulls (and impacts)

With this tool it would be easily feasible to check possible issues when combining, tensions, or see if all is expected to be ok.
Best regards,
Andrea

REPLY 2 (by amarini @ 2026-02-12T10:49:33.245Z):
Hi Andrea,
the information from combine on pulls and impacts are collected in json files. I am not aware of the implementation of the tool that you mentioned (though people did it by hand/eye spotting in the past), but it shouldn’t be too complicated to do.
On the other hand, you should always pay attention to interpret nuisance parameters as poi, eventually understanding if some part of the nuisance is due to different effect in different categories (/analyses/channels).
-Andrea
