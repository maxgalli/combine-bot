Going back to your first question about the saturated GoF per category. This isn't available in the current combine release, but you can try using the code below (*). This will put extra branches in the output tree, one for each category, and one called "constraints", for the constraint pdfs. Note that this is not well validated yet, and not recommended for general usage - you should check carefully that the sum over categories equals the total.
For the checks below - I'm not entirely sure why you get uncertainties of 0.00 coming out - probably something failed in the uncertainty calculation but it's hard to guess without more info. *** Discussion title: Higgs Combination
Hi Andrew,
Thanks for your response. We will start looking into the repo you've
provided, but for you to get an idea of what our approach(es) are: we
use the following GoF command for a single e+ single mu-only fit:
masks_total=SR,dilepton,photon masks
combine -M GoodnessOfFit all_datacards.root --algo=saturated -m 125
--setParameters "$masks_total" combineTool.py -M GoodnessOfFit
all_datacards.root --setParameters "$masks_total" --algo=saturated -t
300 -s 12 -n Toys -m 125 --toysFreq
And we've also tried first creating a workspace with the relevant
regions masked and freezing their rate parameters (without freezing the
rate parameters the MultiDimFit stage fails. These rate parameters are
shared by the dilepton and photon control regions, while the single
lepton control regions share their own rate parameters, which are of
course not frozen here). Command is
freezeparams=dilepton,photon rate parameters
combine .all_datacards.root -M MultiDimFit --saveWorkspace
--setParameters "${masks_total}r=0" --freezeParameters
"${freezeparams}r" -n _MLECRData -m 125
This outputs a higgsCombine_MLECRData.MultiDimFit.mH125.root file which
we then feed into the goodness of fit command instead of
all_datacards.root
(and to study these goodnesses of fit, we run on the output:
combineTool.py -M CollectGoodnessOfFit --input
higgsCombineTest.GoodnessOfFit.mH125.root
higgsCombineToys.GoodnessOfFit.mH125.12.root -m 125.0 -o
goodnessOfFitCROnly.json plotGof.py goodnessOfFitCROnly.json --statistic
saturated --mass 125.0 -o goodnessOfFitCROnlyPlot
which gives us the GoF plots)
The difference between these two sets of commands only seems to affect
the toy distribution, but marginally: the p-value only changes within
~0.1%
