Because of hardware failures in the meantime the data cards of this analysis were lost, however, we could retrieve the corresponding combine workspace. In order to combine with another channel we now have two options:

Convert the workspace back into a data card and run combineCards.py. Is there some functionality in the combine tool that does that rather automatically (basically an inverted text2workspace.py)? Since all the information is still available we can of course write some code that extracts all the numbers and shapes from the workspace, but maybe somebody else already ran into a similar issue?
Combine two workspaces. Does that work, and if yes, how? I couldn’t find any information about that.

REPLY 2 (by amarini @ 2022-05-05T19:01:34.629Z):
Hi Daniel,
Both options are generally little explored in CMS and have pros and cons (many cons).
The first question that I have is if you check the gitlab repositories that we use to save datacards.
HIG does this since run I and datacards are uploaded starting at pre-approval and should be updated whenever a change happens; for HIG, it is an analyzer’s responsibility to make sure that the published results correspond to the latest updated datacard!
which cadilines are they? are you sure that somebody doesn’t have a copy of the datacards somewhere? Finding the datacards is probably the better, easier, and less error prone option.
For HIG, and part of SMP:
https://gitlab.cern.ch/cms-hcg/cadi
I know that other PAGs, e.g. EXO, also have a similar repository.
For what concerns your two questions:

there are no scripts, as far as I know, that converts a workspace into a datacard.
we can probably adapt scripts that navigates the workspaces, but this may depend on how the workspaces were constructed, i.e. starting from which inputs, and the physics model used.
The task may become a little complex. The advantage may be that you have datacards and next time it is going to be easier, and that you can change the interpretation (physics model) you are making.
We have scripts for the LHC Combination, but at the moment they assume that one workspace is from ATLAS and one from CMS. Large adaptations are likely needed and limitations are currently present (last time I tried to do it with two CMS workspaces failed, but maybe it was due to something I was testing). One would still need to double check by hand a set of sanity checks done by combineCards+text2workspace and be aware of a lack of flexibility, potentially check the workspace results by hand. Do you remember which options were used to run text2workspace?

Cheers,
-Andrea

REPLY 4 (by amarini @ 2022-05-06T07:45:58.000Z):
Hi Daniel,
The easiest thing to do is to take the shapes and normalization from the workspace following the text datacard you have.
/afs/cern.ch/user/a/amarini/public/TOP-20-007/rewire.py
This doesn’t check the values of the uncertainties.
From a first check with FitDiagnostics the shapes and data seem to be taken correctly.
You will have to run further checks.
The fit seems to be very close, although it is a bit unstable in both dc.
This may be due to residual differences in how the physics model is applied and/or to differences in the uncertainties, options in the fit (e.g. ranges), or you may have a more up to date version somewhere else.
Cheers,
-Andrea

REPLY 6 (by agilbert @ 2022-08-01T15:52:02.301Z):
Hi Kaitlin,
In theory this could be done, but I don’t think we have any nice scripts to automate it. It also depends on which kinds of uncertainties you want to extract. The lnN uncertainties could be unpacked from the corresponding ProcessNormalization objects in the workspace. The  shape uncertainties are more difficult, partly because the shape and normalization effects are separated in the workspace, but again it could be done by extracting the information from the relevant RooFit objects.
