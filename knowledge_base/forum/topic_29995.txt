Hi Xanda,
the diffNuisances.py script is still in v9 of combine, see here (for v9.1.0) [1]
I think part of the difficulty is perhaps that /test/ is not an often used subdirectory and it is not parth of the path, so it won’t be found from the command line if you just use diffNuisances.py
In fact, there is an open PR which should address this issue (along with some other updates to diffNuisances) [2]/
which we have been a slow to merge; that’s our fault. We will work it into the next version of combine.
Maybe we should also add some documentation for this into the main combine documentation, which the twiki could then link to (which will make it more likely that the B2G instructions dont go out of date or become obsolete).
Further developments of this script for making it more user friendly and flexible for additional use cases are also more than welcome. We’d be very happy to have contributions to that effect, please talk to us (@anigamov and I) if you’d like to work on it; just to coordinate. e.g. developments should probably be made on top of the version in the PR linked above to avoid conflicts.
We can also help with some developments of the script, but the primary limitation in updates and developments is the person power for doing them so if interested parties help contribute that’s the best way to ensure it is done.
Best,
Kyle

REPLY 7 (by ineuteli @ 2025-02-28T11:22:09.596Z):
Hi Hyeon-Seo,
It is in Combine as HiggsAnalysis/CombinedLimit/test/diffNuisances.py. As Kyle pointed out above, it does not appear in the list of commands because it’s in test/ instead of in scripts/. CMSSW only picks up scripts in $CMSSW_BASE/src/<Package>/<Module>/script. When you compile CMSSW with scram b, the python script is not converted to a command. This is documented in https://twiki.cern.ch/twiki/bin/view/CMSPublic/SWGuideBuildFile#CmsswSrcCodeDir.
I typically do something like
python3 $CMSSW_BASE/src/HiggsAnalysis/CombinedLimit/test/diffNuisances.py fitDiagnostics.root -g pulls.root --pullDef diffPullAsym

I guess you could promote it to a command yourself. For example, in BASH:
alias diffNuisances.py="python3 $CMSSW_BASE/src/HiggsAnalysis/CombinedLimit/test/diffNuisances.py fitDiagnostics.root"

although I would advocate of moving it to scripts/ in Combine itself by merging Kyle’s PR #827.
Cheers,
Izaak
