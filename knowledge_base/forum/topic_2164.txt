2. As in my understand, to obtain the breakdown parts for the exp.+theory*, MC sample size, and stat., the procedures could be like the following lines (without scanning of NLL graph):
# ------------------------------------------------------------------------------------------------------------
combine ws\_init\.root \-M MultiDimFit \-n MLUnfold \-\-saveWorkspace \-\-saveFitResult
python3 extractUnc.py multidimfitMLUnfold.root  # It extracts the uncertainty from the fit, \sigma_{tot}
$ mv higgsCombineMLUnfold.MultiDimFit.mH120.root ws_saved.root  # Moved for a convenience
combine \-M MultiDimFit \-d ws\_saved\.root \-n MLUnfold \-\-snapshotName MultiDimFit \-\-saveFitResult \-\-freezeParameters \[\.\.\.\]
python3 extractUnc.py multidimfitMLUnfold.root  # It extracts the uncertainty from the fit, \sigma_{exp.+theory frozen}
combine \-M MultiDimFit \-d ws\_saved\.root \-n MLUnfold \-\-snapshotName MultiDimFit \-\-saveFitResult \-\-freezeParameters allConstrainedNuisances
python3 extractUnc.py multidimfitMLUnfold.root  # It extracts the uncertainty from the fit, \sigma_{all frozen}
# ------------------------------------------------------------------------------------------------------------
Here, extractUnc.py prints, for example, file.Get("fit_mdf").find("r").getErrorHi() (and [...].getErrorLo()), and the printed values are set to be sigma_{tot}, sigma_{exp.+theory frozen} and sigma_{all frozen}, respectively (here I'm abusing the notations of + error and - error).
Also, [...] in the second combine run is the list of the nuisance parameters for all experimental+theory uncertainties which is separated by ',', e.g., jer,jes,lumi,isrttbar (of course it could be the name of group of uncertainty, like exp,theory, with --freezeNuisanceGroups).
And then,
sigma_{exp.+theory}^2    = sigma_{tot}^2                - sigma_{exp.+theory frozen}^2
sigma_{MC sample size}^2 = sigma_{exp.+theory frozen}^2 - sigma_{all frozen}^2
sigma_{stat.}^2          = sigma_{all frozen}^2.
This is the procedures of the breakdown from my understanding. But I'm not sure if it is correct. Could you tell me if there is a problem on this?
3. And also it seems like there is another way to calculate sigma_{MC sample size}^2:
sigma_{MC sample size}^2 = sigma_{tot}^2 - sigma_{autoMCStat frozen}^2,
where sigma_{autoMCStat frozen} is obtained by the following command:
# ------------------------------------------------------------------------------------------------------------
combine \-M MultiDimFit \-d ws\_saved\.root \-n MLUnfold \-\-snapshotName MultiDimFit \-\-saveFitResult \-\-freezeParameters autoMCStats
python3 extractUnc.py multidimfitMLUnfold.root  # It extracts the uncertainty from the fit, \sigma_{autoMCStat frozen}
# ------------------------------------------------------------------------------------------------------------
I'm not sure which way is right. Or, does it depend on my strategy(?) of the uncertainty breakdown?
4. I'm not sure about which category of uncertainties (systematic uncertainty vs. statistical uncertainty) 'MC sample size' is in. Although it is the uncertainty from the "MC statistical uncertainty", it seems like it should be regarded as a systematic one because it is about the model, not about the real data. I also found that some studies classified it as a systematic uncertainty. But I'm still not sure about that.
