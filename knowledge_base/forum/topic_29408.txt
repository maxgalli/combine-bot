REPLY 2 (by adewit @ 2023-09-13T18:24:32.585Z):
Dear Frank,
It’s a bit hard for us to give detailed help without having access to the logs, but it does indeed sound like something in the fit is not going quite right. I’ll try to give some general guidance You can indeed play with the tolerances - note in this case you should be using the --cminDefaultMinimizerTolerance option to set it. The default is 0.1, so to tighten you can e.g. go to 0.01, or to loosen you can go to 1 (but generally good to be careful with loosening this).
The settings you have been changing modify the accuracy of the limit calculation, so it’s not surprising that reducing these values (=increasing the precision) leads to a longer calculation time. However, I suspect this won’t have solved the issue with the error message from Minuit. If this persists, it may be worth trying just a single maximum-likelihood fit (e.g. with FitDiagnostics or MultiDimFit) on an asimov dataset for one of the problematic points, just to check that this behaves as expected (if not, this should probably be the first place to investigate).
With regards to the long job runtimes, you may want to build up a grid of test statistic distributions and calculate the limit afterwards, rather than doing everything in a single job. Some instructions can be found here: https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/commonstatsmethods/#complex-models.
Hope this helps,
Adinda
