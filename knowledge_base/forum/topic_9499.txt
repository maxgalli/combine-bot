REPLY 2 (by nckw @ 2022-04-20T10:22:54.438Z):
Hi Philip,
You should note that the “toysFrequentist” option also tells combine to generate the toy from the profiled value of the nuisance parameters (i.e the nuisance parameters are not randomised), while the global observables (the “means of the gaussian terms”) are generated (from the same values), as explained here : http://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/runningthetool/#nuisance-parameter-generation. You should see for example that the parameters “X_In” are randomised in the toys.
Note also that you might also need to add add the options --toysFrequentist --bypassFrequentistFit also in your last command (eg FitDiagnostics runs) so that combine knows that the toys have different values for the X_In parameters in each toy (though actually it might be that those are automatically picked up these days, maybe another expert can confirm that)
Best,
Nick

REPLY 4 (by nckw @ 2022-04-20T11:48:31.560Z):
Hi Philip,
In general I wouldn’t advise generating the toys with the nuisance parameters on (i.e randomising the constraints) and then fitting with them frozen as indeed thats not ideal. In a very simple example of a counting experiment with one background uncertainty, i’d expect that the fitted value of the signal strength to have a smaller variance in the case of freezing the parameters when fitting (compared to letting them float) since the signal needs to make up for the difference between the data and s+b and in the former case, the b will always be the same (since again, the nuisance parameter itself is generated from the same place, only the constraint term in the likelihood - what we call the X_In – is randomized).
trackParameters should work for any parameter (including _In) so i’m surprised at that. Note that they will be called “trackedParam_X” in the results tree. You can also try adding -v 3 to your commands and check the log that comes from that. It has some details about the toys.
Hope this is helping,
Nick

REPLY 7 (by nckw @ 2022-04-20T13:45:32.480Z):
Hi Philip,
Perhaps one thing you’re missing is that for inference we always (mostly maybe is better) use the profile likelihood ratio. The worsening between the S and S+B fits is also there in the fact that the likelihood has a term (the constraint term) which has a varying central value in the toys → this is what ends up making the inference less certain (r distribution gets wider etc) so its still leading to the same effect that including systematics generally makes things less certain.
All the best,
Nick
