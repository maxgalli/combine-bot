*** Discussion title: Higgs Combination
Dear Brent,
While it is possible to merge workspaces after they are created, it is
probably more trouble than it is worth to do it correctly and then
validate that everything is working - there are various other things
that are needed on top of importing all the objects into the same
workspace, e.g. creating the simultaneous pdf manually.
If you want to use MultiDimFit and are not intending to use
FitDiagnostics with this workspace, I would recommend using the
--for-fits and --no-wrappers options in the text2workspace step for the
combined datacard, which will save a lot of time.
Best,
Adinda

Hi,
I think you need to study the datacards and the model you are using for the combination and understand where the limitations come from. In general, large combination requires a 'little bit' of work in order not to hit limits,
although we try to keep up the code when doing the higgs combination.
You can check the summary at the last Higgs PAG meeting of some of the things we are doing in the full run2 couplings combination to keep things under control, and refer back to the corresponding Higgs Combination meeting for more info. Some of the things there may be experimental and may not work outside of the cards we are using and constantly validating against and on the methods we are not using.
A good place to start is from the beginning. You should consider the following as an hint over a direction to take, and not an actual todo list of commands to run; for instance (not exhaustive), you should start asking yourself:
- what is the overall memory/time for the text2workspace process with the current model and all datacard?
- what is the overall memory/time for t2w with the default model?
- what is the overall memory/time for t2w with each of the datacard in both the default and current model?
- Is there any obvious outliers?
- what are the nature of the datacard you are combining? unbinned, binned-parametric, template?
- for template: do you use autoMCstats for stat uncertainties? does all the cards use a cut for no poisson split? if not is there a reason? how many Poisson parameters are you introducing?
- for parametric: how many RooFormulaVar do you use? (I think this may be a problematic point for you model as well, but I'm waiting for answers to 1. and 2. to say). Is there any obvious place where you are not optimized?
- do you have a lot of process contributing little to the total yields?
