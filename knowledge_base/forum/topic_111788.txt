REPLY 2 (by nckw @ 2025-02-10T12:00:11.903Z):
Hi Federica,
This seems to be an issue in that the parameter you are attaching a flat prior too is one that text2workspace doesn’t know about (yet). basically, we have a null pointer in this block of code : https://github.com/cms-analysis/HiggsAnalysis-CombinedLimit/blob/main/python/ModelTools.py#L834-L843
Note that this doesn’t happen for parameters like normalisations as they can be explicitly present before this code is executed.
I would need to think a bit about how to fix it, unless another expert spots an obvious solution.
Best,
Nick

REPLY 3 (by nckw @ 2025-02-10T13:50:12.192Z):
Hi again,
I thought a bit more and found a workaround. Can you try this PR  https://github.com/cms-analysis/HiggsAnalysis-CombinedLimit/pull/1040  ?
All you need to do in your datacard is to extend the declaration of the flatParam to include a default value (its not very important of course for this) and the range for the prior).  eg,
name flatPrior x [xmin,xmax]
Can you try that and let me know if it works?
Best,
Nick
