REPLY 2 (by agilbert @ 2022-05-06T13:38:20.942Z):
Hi Grace,
In general, this kind of error:
— MultiDimFit —
best fit parameter values and profile-likelihood uncertainties:
Warning - No valid low-error found, will report difference to minimum of range for : r
r : +0.002 -0.002/+0.596 (68%)
Done in 0.00 min (cpu), 0.00 min (real)

happens because the -1σ uncertainty extends below the default lower boundary on the parameter, which will be zero in this case. In principle, you can just fix this by adding something like --setParameterRanges r=-5,5.
In some analyses this leads to another issue (warnings about pdf “underflow”), which is caused by the signal strength becoming sufficiently negative that the total s+b yield in some bins becomes negative, which is a problem for the NLL evaluation (the number of expected events must be positive). But I’d recommend trying the adjusted range first, to see if this gives a more meaningful result.
Note that with -t -1 there are no toys as such, but combine will replace the data in your input with its own Asimov dataset, generated from the default model parameters (with --expectSignal 0 as default). Without -t X, the --expectSignal value is not used.
cheers,
Andrew
