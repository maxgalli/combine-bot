REPLY 2 (by gortona @ 2025-01-16T10:21:11.613Z):
Dear Giacomo,
First of all, let me apologise for sending you on a misleading path. Indeed POIs should be RooRealVars. I thought it was possible to use RooFormulaVar instead but I was mistaken.
I think the only way out in your case is to expand your formulas to resolve the recursivity. What you need to do is write the relations between your parameters so that for a given signal strength you have rXX_yz = f(Rmue,Ratumu,Rtaue,[all other signal strenghts]). At this point this should work.
For example resolving for rdy_tt I believe this is correct (assuming of course I did the math correctly, which might or might not be the case):
rdy_tt = 1/2 * ((rtt_et/rtt_em+rtt_mt/rtt_mm)/2*rtt_mm / rtt_ee * rdy_ee / Rmue /Rtaumu + (rtt_mt/rtt_em+rtt_et/rtt_ee)/2*rdy_ee/Rtaue)

Where the recursivity is gone.
At this point you just need to modify your model removing the lines 58-59 (where you define rdy_tt), uncommenting lines 61-63 (where you create the POIs) and finally add rdy_tt with
self.modelBuilder.factory_('expr::rdy_tt("<formula above >",<all the parameters>)')

Before doing any of this please check if my algebra is correct as I might have done some mistakes.
Best regards,
Giacomo
