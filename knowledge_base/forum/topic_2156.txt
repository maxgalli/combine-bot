REPLY 3 (by Yihui_Lai @ 2021-08-28T01:36:29.000Z):
Dear Adinda,
Thank you for your suggestions!
The best fit value (-12) of p1 comes from using the --trackParameters option in the combine command. We find the reason why the best fit value is different is because the binning of histograms are different.
In the toy data, we used 50 bins, use this as the input to combine, the best value comes from combine is -12. If I change to use more bins, such as 200 bins, the best value becomes -9 and the limit scan becomes smooth again!
So the previous problem seems to come from we are doing a binned fit and the bin width is too large.
Best,
Yihui

REPLY 5 (by andrea_marini @ 2021-08-28T21:59:34.000Z):
Hi,
let me add a couple of points to the discussion:
*) for the binned vs unbinned fits, it is not clear to me what you are trying to do. What we see as “observed” (being observed or a particular dataset is irrelevant) seems to be almost identical between the two plots. If the analysis is unbinned you need an asimov to have the expected results, and you want to make sure that it is created in a suitable way. Here there are some options for Asimov generation in unbinned analyses http://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/runningthetool/#asimov-datasets
*) for binned fits, if your curvature is very large with respect to the bin width the integration performed by roofit is incorrect. See http://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part2/settinguptheanalysis/#unbinned-or-parametric-shape-analysis for more details.
*) in general allowing more parameters, and in particular parameters that modify the shape, will decrease the sensitivity towards the edges.
Did you study the free parameters in the fit? What is the effect on the fitted observable(s)? Do you have eg a node at 500?
Cheers,
-Andrea
