In the combine FAQ, it says:
If you are computing anything else, e.g. p-values, or if your zero-prediction bins are not empty in data, you’re out of luck, and you should find a way to get a reasonable background prediction there (and set an uncertainty on it, as per the point above).

Was the FAQ talking about a single bin in a single background that has a zero prediction, or was the FAQ talking about total background prediction of zero in one bin?

REPLY 2 (by kcormier @ 2024-11-09T08:37:01.111Z):
Hi Jieun,
The documentation is talking about the total expected background being 0. The problem then is that if your model says the total expectation of exactly 0, observing 1 event is simply impossible, so you can’t e.g. compute a p-value for it. Its not a problem if a particular process doesn’t contribute any expected events to some bin.
I don’t think there is any general answer to how to find a way to get a reasonable background prediction in cases where you have exactly 0 predicted events. Methods can include rebinning, smoothing, generating more MC, data-driven estimation etc. But what will work best for you all depends on the problem you’re considering and the constraints you have.
Best,
Kyle
