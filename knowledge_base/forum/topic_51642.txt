REPLY 3 (by anigamov @ 2024-10-08T09:54:02.239Z):
Dear Vinay,
If the categories included in analyses A and B are independent you can combine them using combineCards.py script as described in the docs here. You might want to correlate some systematic uncertainties, and the script will do this automatically if they have the same name, you can also rename systematic uncertainties as described here.
To make sure that covariance matrix is accurate check the fit status stored in RooFitResult within the FitDiagnostics output file as described here. Let me know of something is not clear.
Best,
Aliya

REPLY 5 (by anigamov @ 2024-10-08T11:11:30.163Z):
Dear Vinay,
thanks for following up, I misunderstood what you were asking originally.
Now I see that you want to extract the covariance matrix for expected background for the combination of A and B, and in A you have a CR, while the analysis B is SR-only analysis.
I think it makes more sense to treat A as Type-A and B as Type-B (as defined in the docs) in the combination too, so mask type-A SR only and run the fit with --preFitValue 0 to extract bkg-only covariance matrix. I will let others to comment too.
Best,
Aliya
