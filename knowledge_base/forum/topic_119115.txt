QUESTION (by rseidita @ 2025-03-18T13:34:17.810Z):
Dear Combine experts,
I’m contacting you because of an issue in the context of the EXO-24-010 analysis (search for t-channel production of semi visible jets). We are performing a template fit using ABCD to estimate the shape and normalization of our background, combining 4 categories across the 3 run2 years.
We observe that when adding shape uncertainties on our signal, the order in which we combine the categories seems to change the result of the fit. A minimal example, with only two categories and a single shape uncertainty, can be found here.
We compared the datacards obtained when combining the categories in the two permutations and found nothing obviously out of place. We also tried dumping the contents of the workspaces we get from the combined cards (via text2workspace.py), but could not find obvious differences apart from the different mapping of categories to channels, which is expected and seems consistent.
We are running on combine v10.1.0 under CMSSW 14_1_0_pre4.
Thank you very much in advance for any insight,
Cheers,
Roberto, for the EXO-24-010 team

REPLY 5 (by nckw @ 2025-03-18T17:50:34.000Z):
Hi Roberto,
I suspect it’s an issue when creating the dataset. The code will use the binning from the relevant observable to create an Asimov dataset, whichever version it imported first will then be used to create the Asimov for all channels (since as far as it’s concerned, there is only one observable called MET with 9 bins or 16 depending on the order)
If the names are different, each observable is treated properly
It’s an issue I’ve seen before many times when the ranges are different. Generally it’s safest to use a different name for each category to avoid any issues (there is a warning about this in the docs here https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/latest/part2/settinguptheanalysis/#shape-analyses)
Best,
Nick
