REPLY 5 (by ineuteli @ 2025-02-11T08:33:28.211Z):
Hi @slomte,

Aliya Nigamova:

I believe that channel masks are defined for the saturated model as well, as you can see here. The only difference is that one has to use --setParametersForEval and --setParametersFits options instead.

Note that the options for masking with the saturated model are documented in the manual here: https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/commonstatsmethods/#masking-analysis-regions-in-the-saturated-model

Shivani Sanjay Lomte:

Another question, currently I have the data_obs histogram in the signal regions set as the sum of MC backgrounds to prevent accidental unblinding. Is this okay, or is it recommended to provide the real data and use the -t -1 feature for Asimov?

Probably the masking failed, and you were still picking up the data in the SR in the GoF test, so you get p-value = 1 because you were fitting your B-only model (= sum MC background) to “real observed data” that is actually the Asimov dataset (= sum of MC backgrounds).
Are the data_obs in the CRs of your datacard then actual, real observed data? Someone can correct me if I am wrong, but if all your “observed data” were Asimov dataset, the “observed” test statistic value (blue arrow) would be exactly zero, because the model would fit the “observed” data perfectly. In your case it’s a mix of real and Asimov data because the masking of the SR fails, so the observed test statistic value is not exactly zero, but still below (=better fit than) the toys thanks to the Asimov data in the SR.

Shivani Sanjay Lomte:

Is this okay, or is it recommended to provide the real data and use the -t -1 feature for Asimov?

If you carefully read the manual (here and related FAQ sections) to know what options to use with the different Combine methods, then yes.
For GoF tests, you still need real data. In your case of a GoF test on the CR-only, you simply need to mask the SR channels correctly. If you do not trust yourself or the masking options, I would simply create a version of the “CR-only” datacards, where you have removed the SR channels/bins.
Cheers,
Izaak

REPLY 6 (by slomte @ 2025-02-11T11:44:55.604Z):
Dear All,
Thank you all immensely for your prompt responses and detailed suggestions! The problem is now resolved.
The observed test statistic is falling within the toy distribution.
Upon using real data for data_obs histograms in SR, and using the following options in the GOF command worked just fine.
--algo=saturated --setParameters mask_ch1=1 --setParametersForFit mask_ch1=1 --setParametersForEval mask_ch1=0
I really appreciate the help, thanks!
Cheers,
Shivani
