REPLY 6 (by amarini @ 2022-07-12T13:38:20.000Z):
Hi Sam,
the problem is the definition of the pseudoAsimov for unbinned distribution.
The Asimov dataset is the data observation that corresponds to the S+B hypothesis (or B-only, accordingly)
For what concerns an unbinned distribution, we don’t have a binning where to set the yields, but we can create a dataset that mimics the expectations.
This could be done with different strategies, as explained in the documentation pages.
You may need to further tune it in some places, according to the shape of the pdf distribution and to how large is the pdf parameter space.
you can try to have a larger number of events in the pseudo asimov to have an accurate description of the unbinned part.
Is there still some residual place where --X-rtd TMCSO_PseudoAsimov=5000 is not enough?
If you want to use toys to compute upper limits, that is also a possible alternative.
However, also in that case, the number of toys may need to be large, if fluctuations in the distributions can significantly change the value of the test-statistics.
So you may still need some optimization in this case as well. Is there a reason to believe that the AsymptoticFormula are not valid?
Cheers,
-Andrea

Hi Sam,
I agree, 30% is probably a bit too large to neglect it and 0.1 expected events is indeed very little.
It is also true that 10^6 toys should be fairly enough.
the things than you should try to to are:

make sure that you are effectively running 10^6 toys, and not asking combine to stop early.

make sure that toys have different seeds if ran on different jobs

plot the test statistics distribution (see under complex models in the documentation) and check if it has weird features.

plot the bands vs the poi and check that more or less is continuous (as above).
