REPLY 2 (by agilbert @ 2022-04-25T11:00:59.903Z):
Hi Danilo,
I’m not sure it’s right to think of the GoF test in combine as a “post-fit” (as opposed to “pre-fit”) test as such - it’s true a fit is performed to calculate the saturated model likelihood ratio, but it really quantifies the ability of your model (inc. uncertainties) to describe the observation - and this is usually what we’re interested in. What you are asking for could perhaps be obtained by freezing all nuisance parameters, floating only r in the fit, but then still generating fluctuated nuisance parameters in the toys you use to get the test statistic distribution (so using -M GenerateOnly to produce the toys first). However it’s not immediately clear to me if such a test is useful/meaningful.
Perhaps you can describe in more detail why the normal test is not appropriate here?
cheers,
Andrew

REPLY 4 (by agilbert @ 2022-04-25T12:38:55.265Z):
Hi Danilo,
Just to clarify - the saturated model test statistic doesn’t know anything about the nuisance parameter constraints (i.e. these numbers are not an input to the calculation). It is simply a function of the likelihood, which depends only on the fitted parameter values. Where the constraints come in (indirectly) is via the distribution of test statistic values expected under some hypothesis.
When you say that prefit some variables look perfectly fine, but then get ruled out - how do you define this? Comparing the data points to the prefit uncertainty band by eye, or some other method?
cheers,
Andrew

REPLY 6 (by agilbert @ 2022-04-25T14:06:25.424Z):
Hi Danilo,
The comparison with the prefit uncertainties could give very misleading impressions about whether the model is able to describe the data well. I’ve attached a small sketch as an example. Here we have the data/MC (prefit) distribution for some observable - there’s a clear slope in the trend. Let’s say the only systematic uncertainty I consider is a fairly large, but flat, normalisation uncertainty on the MC prediction (the +/- 1sigma indicated by the horizontal red lines). All the data points are within these red lines, so I might claim this indicates good prefit agreement, but running the saturated model test would tell us that the model is a very poor description of the data - there is no degree of freedom which can significantly reduce the tension between the two.

Datalmc1536×562 47.1 KB

I could imagine that the same kind of effect is happening in your slide - because just looking at the uncertainty band doesn’t tell you about the DoFs and correlations between bins and distributions.
So this is why I worry about the sort of “prefit” GoF test you are asking for - that it will miss genuine problems like this. My gut feeling is that you want to be using the standard GoF test, even if in the end you will use TUnfold.
