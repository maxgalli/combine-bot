REPLY 2 (by adewit @ 2025-12-03T09:46:08.862Z):
Hi Hyeon-Seo,
Is there a reason for all the extra options in your significance calculation command? It’s generally not recommended to add options unless they are needed. From a very quick look, the significances come out very similarly with
`combine -M Significance -d datacard_comb_sig_all_ggh.root -m 125 -n _signif_comb_ggh -t -1 --expectSignal 1 --setParameters pdf_index_ggh=0`
Also note that, assuming your analysis is still blinded, you should not use --toysFrequentist , as it will give you the post-fit expected significance instead of the pre-fit expected.
Best,
Adinda

REPLY 4 (by adewit @ 2025-12-04T13:00:05.972Z):
Dear Hyeon-Seo,
Yes, it’s always better to use as few options as possible, rather than copying from previous analyses (of course if a command-line option is needed for the specific fit you’re running then there’s nothing against adding it, but ideally you can start without options and add the ones you need, rather than the other way around)

hyeonseo:

Since our background is fit with “Envelope” method (having discrete parameters), I believe I would need to keep --cminRunAllDiscreteCombinations  flag. Also since this involves discrete parameters, I think I would need to also keep --cminDefaultMinimizerStrategy 0 option.
Would you agree with my statement above? The rest of the options/flags I can remove if that is your recommendation.

I don’t know the details of when the --cminRunAllDiscreteCombinations flag is needed (it is not always required when using discrete profiling), but there is no harm in keeping it. At worst it will make your fit slower, but it’s not wrong. Minimizer strategy 0 is indeed safer for discrete profiling.

Furthermore, you mentioned that I should not use --toysFrequentist option when I am blinded. I am still using Asimov dataset option -t -1 --expectSignal 1, which would make my significance an “expected significance.” Could you please elaborate a bit further on your statement on this matter?

If you add the option --toysFrequentist a fit will first be done to the data to determine the values of the nuisance parameters which are then used in the generation of the asimov dataset. So you end up with the post-fit expected significance if you add this option, whereas without it you have the pre-fit expected significance, which does not depend on the data at all and is what is considered while the analysis is still blinded. For reference, let me also link the combine documentation on this topic.
Hope that helps,
Adinda
