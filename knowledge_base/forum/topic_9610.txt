REPLY 2 (by agilbert @ 2022-04-25T05:31:35.167Z):
Hi Minxi,
Are the pdfs for all processes modelled using RooHistPdf? If so then I don’t think it makes sense to perform an unbinned fit - a binned fit should be equivalent. And in that case, is there a strong reason to use RooHistPdf over combine’s default implementation (which takes TH1s as input)?
It is possible to use vertical morphing with arbitrary RooFit pdfs - you will need to supply the “Up” and “Down” pdfs in the same way as for TH1-based shape uncertainties (*). For MC stat uncertainties you will also have to construct these by hand as individual shape uncertainties.
cheers,
Andrew
(*) https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part2/settinguptheanalysis/#shape-analysis

REPLY 7 (by minxi @ 2022-04-25T20:45:15.380Z):
Hi Andrew,
The distribution of the bin content is Binomial actually. When the event number is large in that bin,  for example, 5-10, Poisson is a good approximation. And if the number is more larger, the bin content can be assumed distributed Gaussian.
https://arxiv.org/pdf/1007.1727.pdf. In this paper, Fig. 3(a), you can check the difference in the likelihood ratio between the MC experiment and approximation equation with the Poisson assumption.
Then, unbinned shape analysis is necessary for the phase region that lack the events.
And I also think the unbinned shape analysis should have less systemical uncertainty since there is more data point and constraints on the nuisance parameters.
