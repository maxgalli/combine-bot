REPLY 4 (by agilbert @ 2024-03-28T20:28:03.655Z):
Hi Antonis,
The options --cminApproxPreFitTolerance=xx and --X-rtd MINIMIZER_MaxCalls=xx are not really relevant to the problem. There is some issue with the shape of the NLL function at the starting point of the fit. I suspect it’s not only due to the lnN value for Top_Rate_SR4a, rather that the starting point is always on the edge of not having a pos-def Hessian, and small seemingly random changes in initial conditions are enough to make a difference.
I still suspect there is some interplay with the shape parameters, because the unprofiled likelihood scans show some slightly odd shapes (see attached), but you may be able to fix this by moving the start point further from the true minimum (e.g. --setParameters Top_Rate_SR4a=4 seemed to help in a few points I tested).
cheers,
Andrew
