REPLY 4 (by adewit @ 2021-10-29T09:23:07.000Z):
*** Discussion title: Higgs Combination
Hi Peter,
There are indeed other ways to achieve this, though it is unfortunately
not completely straightforward. What you can do is to make use of the
channel masking [*], for which you will need to create a workspace with
this option enabled. Then, you can run a fit on this workspace, masking
the signal region(s) from the likelihood, and saving a snapshot of the
fitted parameters. You'll want to freeze the signal strength to 0, so
you need to redefine one of your rateParams to be the POI:
combine -M MultiDimFit --setParameters mask_ch1=1,r=0 --freezeParameters
r --redefineSignalPOIs <ONE_OF_YOUR_RATEPARAMS> -d ws.root
--saveWorkspace [other options]
(if ws.root is your workspace that you created with the --channel-masks
option, and ch1 is the category that you want to mask)
Now all the parameters of the CR's have been updated to their best-fit
values, ignoring the SR, so the state of the model can now be used for
producing blind limits. You need to run on the output higgsCombine...
file of the previous step, which contains the workspace with snapshot of
the best-fit values. Remember to set mask_ch1 back to 0, otherwise the
SR will still be masked from the likelihood in the limit setting. You
also need to redefine r to be the POI:
combine -M AsymptoticLimits -d higgsCombine.... --snapshotName
MultiDimFit --redefineSignalPOIs r --setParameters mask_ch=0 [other
options]
Best,
Adinda
[*]
https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/nonstandard/#channel-masking
