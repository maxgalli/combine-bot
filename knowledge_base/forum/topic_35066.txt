REPLY 8 (by anigamov @ 2024-02-26T15:22:15.265Z):
Dear Anton,
When you run the FitDiagnostics method by default the signal strength is allowed to float in the range [0, 20]. But it seems like it the best fit point is outside of the allowed range, so the minimum obtained from the initial fit with Impacts method (where r is allowed to float in [-10, 10]) is different from the one you get from the FitDiagnostics.
If you allow the signal strength to vary in the same ranges the fits should be compatible, i.e. try adding --setParameterRanges r=-10,10 to the FitDiagnostics command.
Best,
Aliya
