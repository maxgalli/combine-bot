REPLY 2 (by klin @ 2025-05-08T16:43:47.709Z):
The anaysis @yilai can reproduce most of the limit when the signal mass MX is below 3.5 TeV:
Screenshot 2025-05-08 at 18.16.54906×732 53.7 KB
As this published analysis is second strongest channel in the combined limit for a Drell-Yann produced heavy resonance, I didn’t want to simply accept this difference. The interesting feature is that observed limit is mostly below the expected limit, which can be explained by the post-fit distribution in the signal-mass observable.[1] But the two close in for the highest mass in the original analysis. In contrast, the current result by the analyser of the combination saw them far apart. My own running of Combine saw the same results.
This let me start to question the fit stability. I tried to use different random seed but the likelihood scans in r didn’t change at all. I also saw that the scan couldn’t find the global minimum in r: data is below nominal background so the fit drew the signal negative, but before the NLL flatten it ran into negative values of PDF so RooFit broke at around r=-0.35.[2]. But this shouldn’t cause any issue for the expected limit because their the Asimov data is created by a fit where r is fixed to 0 and the test-statistics, \tilde{q}, doesn’t allow \hat{r}<0.
I also wonder if the old analysis used toys to produce limits because the data there certainly is Poissonian. But then due to the big gap between data and nominal-background, 2000 toys of signal hypothesis and 500 toys of Asimov hypothesis can hardly find any \tilde{q} above the observed \tilde{q}.
qmu_MX4500_r0.2.pdf (21.4 KB)
I haven’t tried median expected using toys but this led me to suspect such possibilities.
The original analyser has switched topics and the paper was published 3 years ago. So I am not sure if we can retrieve more info to cross-check.

This analysis has two observables, one for resonance mass and another for the jet mass. ↩︎

The default r=1 corresponds to 1 pb signal cross-section so the analyser added and fixed a rate-parameter that scales it down to 1 pb ↩︎

REPLY 5 (by adewit @ 2025-05-08T18:44:03.048Z):
Hi Kuan-Yu,
If you are getting different results even with the version of Combine that was used to produce the analysis then I really think you need to discuss with the original author team whether the datacards/model you have really is the latest version, and whether any particular settings were required. I don’t know if you’re sure of the version that was used at the time - in 2021 the recommended version was some version of v8, but it wasn’t routinely checked that people were using the latest recommended version. The original analysts may still be able to provide this information.
I think there is only any debugging to do once you are certain that there is a Combine version in which you can reproduce the published limits - otherwise it sounds like you don’t have the version of the datacards that correspond to the publication.
Best,
