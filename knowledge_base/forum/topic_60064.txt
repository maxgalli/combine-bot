We have 120 Signal Regions (SRs) and 6 Validation Regions (VRs) for each Run2 data taking year (2016_APV, 2016, 2017, 2018). A region is defined as some combination of four hadronic jets tagged as W, Z, Higgs, top, bottom, QCD. The regions with 3 or 4 QCD tagged jets are the unblinded Validation Regions. Each region, in each year, is a datacard with a single channel.
Each region has signal MC, DataDriven QCD background, and various contributing background processes.
I am using Combine v10.0.2, CMSSW_14_1_0_pre4.
We produce ROOT Ntuples using CMSSW_10_6_29 (Run2 UL recommendation), but preprocess those ROOT files on CMSSW_14_1_0_pre4.

REPLY 9 [ACCEPTED ANSWER] (by adewit @ 2024-11-22T08:55:37.250Z):
Hi Samantha,
From having a quick look I don’t see anything obviously wrong with your fit, it just looks like you have very small numbers of background events in some of the bins, and then for negative values of the signal strength mu*S+B in some of the bins becomes <0, thus leading to the FASTEXITs that you see.
For the impacts, I think you have two options:

Set --rMin to 0. The impacts routine should still work, but all the impacts will be one-sided (you nevertheless get an idea of which uncertainties are important, of course)
Evaluate the expected impacts for a positive value of the signal strength (e.g. the value of the expected limit, or some other choice).

I note that while running your fit, I wasn’t able to get sensible results for the initial fit in option 1) with the signal scaled by a factor 1000, so I would use the version where you do not scale the signal for future checks. If I set this scaling back to 1 I see that the crossing for the 68% confidence interval is at around 0.8. The ‘no valid low-error found’ that you see in your output is related to this - if you scale by 1000, you’d expect the crossing to be at 0.0008 and the +ve range of +3 is too wide for the crossing to be found (and even if you reduce the range by a factor 1000, Combine only reports 3 digits after the decimal point, so you would still not be able to get an accurate interval from this).
I hope this helps a bit!
Best,
Adinda
