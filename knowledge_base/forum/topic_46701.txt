QUESTION (by mrogulji @ 2024-08-26T09:22:53.411Z):
Dear Combine experts,
I’m trying to set upper limits on a cross section at 95% CL and would like to compare Asymptotic and FeldmanCousins methods. If I understand correctly from the example, to calculate the observed limit for FC, one would need to run single point scans at several cross section values and see where the test statistic crosses 0.05. How do we get the expected value? Do we throw N toys for each scanned point and see the cross section for which the median test statistic crosses 0.05 (and the same for the uncertainty band, after adjusting the quantile)? Thanks.

Matej

REPLY 2 (by kcormier @ 2024-08-27T09:18:35.198Z):
Dear Matej,
Yes, it sounds like your intepretation is correct. Since you are comparing to an asymptotic limit, it seems you are fitting a single parameter, in which case combine can do the interpolation for you, as explained here [1]. Though, if your goal is simply to test the asymptotic approximation, you could use the machinery for calculating limits with toys [2] – theres not really a reason to use FC specifically in that case. Depending on what specifically you’d like to test by comparing to Feldman-Cousins there may be other easier ways of doing things as well.
But if you’d really like to check the Feldman-Cousins result with toys, you can get the expected results by generating toy data as you said. I don’t think there is a built-in way to do this automatically like what exists for the typical upper limits. If I’m forgetting it, hopefully someone else will jump in.
Best,
Kyle
[1] https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/commonstatsmethods/#extracting-1d-intervals
[2] https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/commonstatsmethods/#computing-limits-with-toys

REPLY 4 (by kcormier @ 2024-08-27T11:05:11.399Z):
Dear Matej,
This shouldn’t be a problem. The relevant grid results are stored in a RooStats:HypoTestResult object which is under the toys directory, and should contain all the POI information necessary.
If you want to include the POI value in the output tree you can use the --trackParameters <paramName> argument [1] to store any parameter you’d like in the output tree. But this shouldn’t be necessary for the method.
Best,
Kyle
[1] https://cms-analysis.github.io/HiggsAnalysis-CombinedLimit/part3/runningthetool/?h=trackpar#common-command-line-options
