QUESTION (by anikiten @ 2025-04-10T09:13:11.164Z):
Dear experts,
I run combine command:
combine -M MultiDimFit 2018UL_dimuon_SR1.txt --algo fixed --fixedPointPOIs r=0 --redefineSignalPOIs r --robustFit=1 -v 3 --rMin 0 --rMax 120
and I see it does first, fit S+B  and second, fit B.  I have two questions here:

where the second fit (B) takes the initial values for background
parameters from ? Is it bkg. parameters taken from S+B fit values or
it my initial values (as S+B fit) ?
Is there an option to change order of fits: first B, and second S+B ?

Thanks, Sasha.
----------------------------------------

REPLY 2 (by gortona @ 2025-04-10T10:09:40.000Z):
Dear Sasha,
When using the fixed algorithm of MultiDimFit, it will indeed perform two fits, but they are not S+B and B. The first one will determine the best fit floating the POIs, and the second one will fix the POI to the values chosen by the user. It appears like being B in your setup since you have chosen the r=0 point.
I’m not 100% sure but I think the starting values for the parameters in the second fit are the final values of the first fit. In any case it should not be much relevant since all the parameters are fitted again under the new POI assumption so it should only be affecting the computing time.
To my knowledge there isn’t any command in combine allowing to change the order in which the fits are performed, but I don’t think it should make any difference.
Cheers,
Giacomo

REPLY 5 (by gortona @ 2025-04-10T12:30:14.000Z):
Dear Sasha,
I’m not sure I agree that there is a dependency.
When running the second fit you in any case perform a fit of all the parameters to the data (or pseudodata). Barren small numerical sensitivity differences or very complex models with multiple minima, the initial values of the parameters should not affect the post-fit values you get. Therefore the order you run the fits it doesn’t actually matter.
And even in the extreme case I was mentioning before, you most likely are interested in the minima closer to your best fit evaluation, so using that set as initial value it’s actually beneficial.
Cheers,
Giacomo

REPLY 7 (by gortona @ 2025-04-10T13:36:40.000Z):
Hi Sasha,
Let me try to be a bit clearer, because I feel there might be a misunderstanding about my usage of multiple minima.
When you run, you will find the global minimum of the likelihood function with the first fit, where all POIs are floating.
Then, in the second run you will fix the POI to a given value and find the local minimum for this profiled likelihood, which will be different from the global one. If the profiled function is well behaved, you will always find the same local minimum regardless of the initial values for the parameters.
The issue I was mentioning before when speaking of multiple minima arises only if you have multiple minima in the profiled likelihood function for a given POI. Is that what happens in your case?
One way to test this could be to run at r=0 with your initial values and with those found at the global minimum and check that you actually obtain different results (beyond numerical differences).
Cheers,
Giacomo

Hi Sasha,
If that’s your case then I think your way out is to save a snapshot for the best fit point and then when you call the MultidimFim —algo=fixed you can load the snapshot and add the —skipInitialFit option. Hi Sasha,

When you run, you will find the global minimum of the likelihood function with the first fit, where all POIs are floating.

Giacomo Ortona gortonaApril 10

Hi Sasha,
Let me try to be a bit clearer, because I feel there might be a misunderstanding about my usage of multiple minima.
When you run, you will find the global minimum of the likelihood function with the first fit, where all POIs are floating.
Then, in the second run you will fix the POI to a given value and find the local minimum for this profiled likelihood, which will be different from the global one. If the profiled function is well behaved, you will always find the same local minimum regardless of the initial values for the parameters.
The issue I was mentioning before when speaking of multiple minima arises only if you have multiple minima in the profiled likelihood function for a given POI. Is that what happens in your case?
One way to test this could be to run at r=0 with your initial values and with those found at the global minimum and check that you actually obtain different results (beyond numerical differences).
Cheers,
Giacomo
