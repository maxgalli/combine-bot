QUESTION (by gmelachr @ 2025-10-22T19:19:15.100Z):
Dear Combine Experts,
for the BPH-24-005 analysis we are trying to use the COMBINE package in order to do some simultaneous measurements. In order to save some space, we summarized what we are trying to do in this
combine_setup_BPH_24_005.pdf (117.1 KB)
pdf, which includes texts from the latest version of the AN, which is not published yet.
BPAG has done similar analyses, however, none (or at least I am not aware of any) used COMBINE. For example, in the BPH-22-005 analysis, where we measured the dB/dq^2 for the B+ ->μμΚ+ decay mode, we fitted independently the mass distribution for the J/ψ bin and for the rare mode using simple roofit, we calculated a*e from the MC and inserted everything in the formula. For the systematic uncertainties, we assigned corrections on the MC and we recalculated the ae factors, we varied the corrections up and down by 1σ and we used the maximum difference as the systematic uncertainty.
But for the BPH-24-005, the idea is to use COMBINE in order to calculate directly the dB/dq^2 observables and assign systematics to these observables. So, we ended up with the procedure that is described in the pdf. Please find attached a datacard
mumuLambda_11_rare_Jpsi.txt (6.8 KB)
and the workspaces that are created by the roofit code
roofit_workspace.txt (21.5 KB)
and the text2workspace command
text2workspace.txt (33.8 KB)
This procedure has been proven though very difficult and challenging and till some point it is working, however there are still many issues. For example, if we just define the normalization for the J/ψ bin as model_LambdaSignal_LambdaJpsi_norm, then the text2workspace command creates the shapeSig_signal_LambdaJpsi__norm variable and assigns it for the normalization of the signal process, but it is not assigning the same variable for the RooFormulaVar which is needed for the q^2 bins. Therefore, instead of creating the model_LambdaSignal_LambdaJpsi_norm variable, we created the shapeSig_signal_LambdaJpsi__norm by ourselves in order to make it work and use the same quantity for both the J/ψ bin and the q^2 bins. But this trick still creates some issues, since it changes the roofit result
simultaneous_Lambda_Jpsi696×750 18 KB
to this prefit distribution.
part2_sb_model698×476 6.28 KB
Since we do not want yet to sacrifice our efforts with COMBINE and follow a more BPAG-“traditional” strategy, we are seeking for a way to avoid the RooFormulaVar’s (which we think that is resposible for all this chaos) for the normalizations of the q^2 bins and we are asking for your feedback and your guidance in order to build a new physics model. Do you think that it would be possible to make COMBINE calculate by itself the dB/dq^2 observables by using the RooRealVars normalizations from all the bins, omit the signal strengths “r”, read somehow the a*e factors (from the datacard maybe?) and calculate the dB/dq^2 observables with their statistical and systematic uncertainties? Do you know if there is a physics model that could be used for such a purpose and if not, where should we start from?
Please let us know if you need more explanations.
Best regards,
Giorgos for BPH-24-005

REPLY 3 (by nckw @ 2025-10-22T20:33:02.000Z):
Dear Georgios,
This certainly sounds like a good candidate for a Physics Model to be the solution. Also this will help preserve setups for possible future analyses.
In principle from your pdf, you should be able to create parameters that are dB/dq^2 and express the yields in different channels in terms of those - it’s a rearrangement of your first equation.
Here is an example from the Higgs world : https://github.com/cms-analysis/HiggsAnalysis-CombinedLimit/blob/45b037b2a3a5e8e85da7ca3ebcede9dbc5480db7/python/LHCHCGModels.py#L605 which we use when we want to measure ratios of Higgs couplings.
When you do this, the systematics on efficiencies etc should also be expressed as uncertainties on those yields. It means you don’t need to calculate the uncertainty efficiencies directly, but rather calculate them for each channel and the fit will handle cancellation provided correlations are properly encoded.
I may have oversimplified things and missed some subtleties so I’d be happy to discuss with you and others in case the example above doesn’t clarify what’s needed / if I’ve missed some reason why it won’t work.
Best,
Nick
