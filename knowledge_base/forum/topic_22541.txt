This work should be pretty much in line with TOP-21-001 where Wilson coefficients can alter the distributions.
Let me outline the list of objects that I have right now or I can generate:

Background mountain range distributions from ttbar and ttH processes, which could be represented into one shape histogram or several histograms.
Signal (four tops) mountain range distributions, generated at LO with different values of yt (which I want to measure). This can be represented as different sets of histograms per each yt value, which should be normalised to the theoretical cross section value as a function of yt. The yt values used to generate these distributions include the SM value.
Histograms representing shape uncertainties for background and signal processes.
Signal (four tops) mountain range distribution, generated at NLO from a central sample. I originally plan to use this as a signal distribution, scaled to the measured cross section value of 17 ± 5 fb, as reported in TOP-21-005.

Here’s what I have in mind to measure the value of yt from these objects:

For each yt value, construct a datacard containing signal and background distributions. These datacards should have the same set of both normalisation and shape uncertainties for background processes, and same normalisation uncertainties for signal processes. Shape uncertainties for signal processes should be different for each yt value.
As this is a work in progress, I would like to start small by constructing an Asimov dataset based on four tops distributions with yt = SM. (I may use the same method with four tops NLO sample, scaled with measured cross section as well.) Based on this Asimov dataset, calculate the likelihood value (or -2 ln L) based on each yt value.
Based on -2 ln L values from each yt value, measure the best probable value of yt using NLL procedure.

The problem that I have with this procedure right now is that the distributions change with different values of yt, and there’s no guarantee that the changes are linear like a conventional signal strength. Furthermore, since the distributions are different depending on yt, they have to be represented in different datacards. As far as I checked in the Combine documentation, there is no direct way to calculate the likelihood value for each yt case, compare them, and make measurements manually as I am proposing.

REPLY 2 (by kcormier @ 2023-04-11T12:28:13.078Z):
Hi Vichayanun,
Given that you are trying to measure yt, it sounds like you would like this to be your parameter of interest in the fit, rather than setting up datacards to perform fits for different values of yt.
Since yt is your parameter of interest, you probably want to construct a PhysicsModel class which properly handles this. The PhysicsModel class defines the parameters of interest (POI), as well as how they scale process yields. The default model is a simple one in which all signal processes scale linearly with a single POI, “r”. But there are others. For example there is a MultiSignalModel which is defined and can scale different processes with different POIs.
But the models don’t need to scale linearly with the POI. This can be defined by the PhysicsModel class. A good place to start is probably looking at the Phyics Models section of the documentation [1].
There are existing PhysicsModels within the code which were already developed for many Higgs Analysis, and there are examples of how these are set up and used in some of the previous combine tutorials, where there are specific presentations on PhysicsModels such as [2]. As you can see, its possible to define a PhysicsModel handling many cases, beyond simple linear scaling of a signal with the POI.
Have you considered defining a PhysicsModel to handle your case? If there’s some reason this doesn’t work for you, let us know.
Best,
Kyle

However, I also need to model uncertainties which are dependent to yt, such as JES, so I decided to implement it in the form of rateParams in the same way as in TOP-21-004 analysis.
My approach for the implementation is that, since it is a shape uncertainty, I should follow the interpolation described in Higgs Combine documentation [1], but since each datacard now has one histogram bin, f(\theta=0) should be 1. Since the nominal/upper/lower templates can be fitted to polynomials, I can write the yield of each bin in the form of P_0(y_t), P_+(y_t), and P_-(y_t) respectively.
Based on the interpolation formula, I should have \delta^+ = P_+(y_t)/P_0(y_t) - 1 and \delta^- = P_-(y_t)/P_0(y_t) - 1, and from these parameters I define the rateParam with the structure of
1 + (((delta_p-delta_m)*theta + (3*theta^6-10*theta^4+15*theta^2)/8)/2) * (theta < 1 and theta > -1) + delta_p*theta * (theta > 1) + delta_m*theta * (theta < -1)
Here is the snippet I used to generate such rateParams:
tttt_fitting = polynomial_print(coeffs)
tttt_fitting_unc = {}
for key in fourtops_fitting_unc.keys():
tttt_fitting_unc[key] = polynomial_print(coeffs_unc[key])

combine_card_customparams += f"{binname}_ttttrate       rateParam       {binname}  tttt  {tttt_fitting} yt\n"
for unc in unc_names:
deltap = f"({tttt_fitting_unc[unc+'Up']})/({tttt_fitting})-1"
deltam = f"({tttt_fitting_unc[unc+'Down']})/({tttt_fitting})-1"
shapeunc_formulation = f"1+(((({deltap})-({deltam}))*@1+(({deltap})+({deltam}))*(3*@1^6-10*@1^4+15*@1^2)/8)/2)*(@1>-1&&@1<1)"
shapeunc_formulation += f"+({deltap})*@1*(@1>=1)+({deltam})*@1*(@1<=-1)"
if unc in ["jeslowess", "jerlowess"]:
combine_card_customparams += f"{binname}_ttttrate_{unc} rateParam {binname}  tttt  {shapeunc_formulation} yt,{unc}\n"
else:
combine_card_customparams += f"{binname}_ttttrate_{unc} rateParam {binname}  tttt  {shapeunc_formulation} yt,tttt_{unc}\n"
