REPLY 6 (by gortona @ 2024-10-11T12:04:28.464Z):
Dear Shivani,
I think the origin of the problem is that in your 2016 workspace the observable is called rather awkwardly min(999.9999,met)_monohiggs.
When we have more than one observable, they are accessed by splitting the list of all observables by commas, so the comma in the middle of the variable name in this case breaks everything.
I’m not sure there is an easy fix to your case. If possible, the best solution would be to recreate the 2016 workspace with a different observable name, such as met_monohiggs. Otherwise, you can try to hack the code here:

github.com

cms-analysis/HiggsAnalysis-CombinedLimit/blob/main/python/ShapeTools.py#L599

self.out.mode = "unbinned"
if self.options.verbose > 1:
stderr.write("Will work with unbinned datasets\n")
if self.options.verbose > 1:
stderr.write("Observables: %s\n" % str(list(shapeObs.keys())))
if len(list(shapeObs.keys())) != 1:
self.out.binVars = ROOT.RooArgSet()
for obs_key in shapeObs.keys():
# RooArgSet does not have method to add another RooArgSet (RooCollection does!) so we have to manually loop over the RooArgSet contents (in obs)
if "," in obs_key:  # if multiple vars and looking at a RooArgSet value
for k in obs_key.split(","):
self.out.binVars.add(shapeObs[obs_key].find(k), True)
else:
self.out.binVars.add(shapeObs[obs_key], True)
else:
self.out.binVars = list(shapeObs.values())[0]
self.out.safe_import(self.out.binVars)
else:
self.out.mode = "binned"
if self.options.verbose > 1:
stderr.write("Will make a binned dataset\n")

and see if you manage to make it work with an ad-hoc solution.
I hope it helps,
Best regards,
Giacomo
