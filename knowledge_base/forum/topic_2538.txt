REPLY 2 (by amarini @ 2021-11-18T20:50:24.000Z):

Hi,
"Systematics which have different names will be assumed to be uncorrelated, and the ones with the same name will be assumed 100% correlated."
In your case, you can use the same parameter name for the scale uncertainty in all the categories. For instance, if you have as signal model a gaussian distribution in the two (or +) categories, it is convenient to separate the constraint part to its value (and sign for anticorrelation); a multiplicative gaussian constrain on the scale can be written as follow:
[ws content]
mu1= mu1_value[125.1] * (1+scale_value1[0.03]*scale)
mu2= mu2_value[124.9] * (1+scale_value2[0.06]*scale)
[*] the only floating variable above is 'scale'.
you can use
[datacard content]
scale param 0 1
to correlate both the categories to the common constraint scale.
