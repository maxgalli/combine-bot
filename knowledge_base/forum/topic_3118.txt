REPLY 2 (by adewit @ 2021-12-09T14:46:28.752Z):
Dear Maryam,
If you open up the file produced by FitDiagnostics (fitdiagnostics.root), and call “fit_b->Print()” and “fit_s->Print()”, do you also see 0±0 for all these parameters? That will tell you if there is a problem with the fit or with the script. If you also have 0±0 for these parameters when you print from fitdiagnostics.root directly, something might have gone wrong when running FitDiagnostics. Did you see any error messages when you ran it?

REPLY 5 (by adewit @ 2021-12-14T13:11:39.942Z):
Dear Maryam,
Ok, something is not working well in the calculation of the covariance matrix – in your printout I see
“covariance matrix quality: Not calculated at all”, so that’s why you’re getting these strange results. You can run FitDiagnostics with the option -v 3 to get a more detailed error log from the minimizer. I don’t know exactly how you’re constructing your background model, but your AN mentions you’re using the discrete profiling. If that is the case then the covariance matrix calculation in FitDiagnostics will never work, so then I think this part of the checks should be ignored.
Best,
Adinda
