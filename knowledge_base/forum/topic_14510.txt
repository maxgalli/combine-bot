Some small suggestions:
In the last fit, you might try to increase the parameter range for r_H_PTH_GT600. Also worth heeding the warning to set starting values for the POIs (even if this is a data fit) with -setParameters
As a note, for combine v8.2 (102x) and up, –X-rtd MINIMIZER_analytic is implied.

Hi Abdollah,
From what you pasted above, there is still a failure to determine the uncertainties on r_H_PTH_GT600:

Usually at this stage one has to turn up the verbosity (-v 3) and try and get clues from the Minuit logs as to why the fit is not converging. We have some slides that cover some the debugging steps: https://indico.cern.ch/event/976099/contributions/4138476/attachments/2163625/3651175/CombineTutorial-2020-debugging.pdf
If you prefer not to do this yourself, we can try and help, but please share the exact workspace you are using, and give the full combineTool.py command to reproduce. Note that understanding these kinds of problems can be very time intensive - we may not be able to respond quickly.
