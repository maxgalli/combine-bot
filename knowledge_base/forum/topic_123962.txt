REPLY 6 (by gortona @ 2025-04-23T19:34:37.000Z):
Dear Sasha,
The problem is that, as explained in another thread, what actually makes (statistical) sense is a likelihood ratio, not the likelihood value per se.
Since we assume we always use likelihood ratios, there are constant terms that are neglected in the minimisation procedures both in combine and roofit, so we don’t actually know the “real” nll value, nor does roofit.
Regarding your original question, I believe this sequence of steps should be working:

create a best fit snapshot
run with the grid algorithm tuning it to get a 1-point grid with r=0 + load the snapshot + use the skipinitialfit option + set the model parameters to values of your liking

If I’m not mistaken sincr the snapshot js loaded before anything else and you skip the first fit, this should let you have the r=0 fit with your initial parameters.
Cheers,
Giacomo

REPLY 9 (by gortona @ 2025-04-28T09:48:32.793Z):
Hi Sasha,
As I said in an early reply, modify the fit order is not at all easy, and this is a very peculiar user case. I don’t think we have the personpower to make such a change to the code.
At the same time, we don’t want to print the NLL value since we can’t ensure it is going to be used consistently.
About your questions the logic is the following:
What you want is a way to perform only the r=0 fit from a pre-defined starting point and compare the result to the best fit value.
The snapshot will contain the global best fit value of your POIs and the values of all NP at the best fit. So once you can load your best fit snapshot you will have the best-fit reference point. Since the snapshot is loaded before any other combine command, by using setParameters you can restore your desired starting parameters values. Finally, you don’t want to re-run again the S+B fit, otherwise you would be again resetting your NP starting values for the r=0 fit (since S+B is ran first). Hence the need for the skipInitialFit command. This way you will only run r=0 with the chosen parameters.
Finally, about the usage of grid, I’m sorry this was a mistake on my side, skipInitialFit should work with the fixed option as well.
I hope it is clear,
Best regards,
Giacomo
