Hi all, I have some confusion about how to use the --setParameters option to initialize several flatParams in my fit to a default variable.
For some background, I’m running a custom framework built around Combine which allows for two-dimensional maximum likelihood fits. I run the following command:
combine -M FitDiagnostics card.txt --text2workspace "--channel-masks"  \
--saveWorkspace --cminDefaultMinimizerStrategy 1 \
--rMin -1 --rMax 5 -v 2 --robustFit 1

My datacard and the file from which it reads are attached [1, 2] and the card contains flatParams at the end, some of which I’d like to initialize to a specific value:
...
Background_SR_loose_HIGH_bin_2-1          flatParam

I attempted to do this by appending the following to my Combine invocation:
--setParameters Background_SR_loose_HIGH_bin_2-1=10.0,[etc...]

REPLY 2 (by anigamov @ 2024-01-22T14:13:50.318Z):
Dear Amitav,
I think  --setParameters Background_SR_loose_HIGH_bin_2-1=10 does what one would expect it to, but in the table which is printed by RooFit the values in the InitialValue column are not really initial values that you set, they are coming from an intermediate step in the minimisation routine.
It looks like you have many of those flatParam’s  at the boundary after the fit, perhaps it would be good to check the inputs again, and make sure that the models works without fitting it to data, i.e. adding the -t -1 option
Best,
Aliya
