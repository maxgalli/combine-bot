REPLY 2 [ACCEPTED ANSWER] (by kcormier @ 2024-06-19T10:31:37.318Z):
Hi Bill,
Sorry for the delayed response. I took a quick look, and I think the issue is related to the order in which things are being declared in your datacard.
It seems you have some RooFormulaVars defined which rely on params (or rateParams) which aren’t defined until later lines in the datacard, I think this is what is causing the fault.  (see for example, the FormulaVar defined on line 181 which references rateParams defined on lines 235 and 219).
CombineHarvester is parsing the formulaVar line first and crashing because the relevant parameters aren’t in the workspace yet. There are some other lines like that too, so you’ll need to fix them all.
Then, unfortunately it seems there is another problem which is a bug in the current recommended release of CH (v2.1.0) however, it has been fixed in the main branch, so you could use that. There will also be a new CH release soon which will include the fix.
Hope that helps,
Kyle
