REPLY 2 (by amarini @ 2022-11-09T10:56:35.672Z):
Hi Sam,
it is unclear to me what you exactly want to do with the decorrelation, if you are trying to propagate errors from another fit or just easy the fit convergence.
RooFit will not compute eigen values/vectors because these quantities are defined after the fit. You can choose a point in the parameter space, compute the hessian matrix, and rotate the basis of the parameter is such a way that the hessian is diagonal. If you are at a local minimum of the log-likelihood, this means that the covariance matrix is also diagonal. However this transformation is (in general) point-wise dependent.
A reason for doing it in anycase at some point and with some set of uncertainties could be to remove a large part of correlation/anti-correlation of some parameters and therefore easy the fit convergence.
This means that if you had a probability density function P(a,b,c… ) now you can write it as P(alpha, beta, gamma, … ) where a, b, c  are linear combination of alpha, beta, gamma.
A reparametrization of this kind does not change the likelihood function nor it’s capability of describing the dataset, but has only numerical advantages in the minimization process.
There is no particular treatment in combine wrt different parametrizations. You will simply use the new pdf instead of the old one.
If you have another fit from which you want to propagate some uncertainty as external measurement,
you will diagonalize the result wrt that fit and add a constrain term for the eigenvector of the covariance matrix of that fit. A param statement add this constrain terms to the likelihood function.
(Are you sure this is what you want to do?)
