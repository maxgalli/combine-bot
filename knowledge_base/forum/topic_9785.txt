QUESTION (by cgrimaul @ 2022-04-27T18:01:54.788Z):

We are performing a shape analysis of 1D binned histograms to extract the tau lepton polarisation from data. And also another concern: is it sufficient to run the fits on the combined cards for AD and KS in the aim to extract results from individual channels ? REPLY 2 (by adewit @ 2022-04-28T08:32:11.326Z):
Dear Clement,
It looks like the images you tried to attach aren’t shown on CMStalk, nevertheless I think most of your questions can be answered without them.
Regarding the  use of --toysFrequentist with the KS and AD algorithms: To be honest I don’t remember the reason for generally preferring default toys with these algorithms, maybe one of the other experts remembers and can shed some light. In any case, if you use this option the outcome should still be valid (in general the argument is that the exact way in which the toys are generated should not affect the test statistic distribution, but there can clearly be cases where this is not true). If you want “pol” not to be fixed to the default however, you should make sure that it is a standard floating parameter rather than a POI, since POIs are fixed in the fit to data when you generate toys with --toysFrequentist. I think calling --redefineSignalPOIs r, assuming that is the name of the signal strength parameter you use, should be sufficient to achieve this, but it’s always good to keep an eye on the fit output to be sure.
About running the fits on the combined cards to extract results from individual channels, I’m not sure I fully understand the question, in particular I think I don’t understand what is meant by the “combined polarisation” of all categories. The KS and AD tests are defined at the level of each individual category, but the values of the fitted parameters used to evaluate the test statistic all come from the same fit.
Finally, if you find strange test statistic values in some of the toys, it’s probably best to look at those toys in a bit more detail to see if anything odd is happening, e.g. increasing the verbosity (-v 3) and inspecting the output logs.
Hope this helps,
Adinda

REPLY 5 (by adewit @ 2022-05-04T10:33:54.658Z):
Dear Clement,
Indeed it’s not a trivial question - what I can say is that the tests are all sensitive to slightly different effects - for example KS and AD should be similar, but AD gives extra weight to the tails, so discrepancies in that region might lead to a worse compatibility in this test than with the KS test. Similarly I could imagine the per-region saturated model test to behave better if you have many correlated parameters for which there is some tension between the categories (because in the saturated case to get results per-category you are only fitting one region at a time)
For the hypothesis about the AD contributions - I’m not sure you can conclude this. When you change the binning the test statistic distribution generated with the toys will also change - not just because the binning itself is different but also because the nuisance parameters might have different fitted values.
Best,
Adinda
