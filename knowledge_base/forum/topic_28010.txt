REPLY 2 (by kcormier @ 2023-08-08T15:31:04.088Z):
Hi Stephanie,
Yes, the asymmetric Poisson errors are expected. As with all confidence intervals, there is some freedom of choice in how the interval is defined, but I think these are the commonly used intervals and agree with the intervals given in the pdg review of statistics [1] for poisson data. You can check the expected error values e.g. using python [2], and should see that they match what you are getting. So I think this is all expected based on the standard confidence interval construction methods we are using, and I don’t think there is anything that combine is doing strangely here (in fact I think the error values you are seeing are constructed by ROOT, not directly by combine).
As for the effect that this has on the fit, the confidence intervals themselves will not have any effect, as the Poisson error bars, as well as the error bands drawn for the prediction are not used directly in the fit, they are only “summarizing” (and providing a visualization) for the information. When the fit is actually performed it uses the full likelihood, and any confidence intervals (or uncertainties/errors more generally) are constructed from the likelihood used in the fit, not the other way around.
The likelihood does use a Poisson term, and so your observation about the asymmetry is correct in the sense that for some observed count, the Poisson portion of the likelihood for the “true” number of expected events being n+b will generally be higher than that for it being n-b, and the fit will not treat them equally. But this is exactly because the probability of observing n events when the “true” expectation is n+b is higher than the probability of observing n counts when the “true” expectation is n-b.
I don’t think I fully understood what you are asking when you ask if we are increasing the probability to observe an upward fluctuation when the fit is made; If you’d like to clarify I’ll follow up. But as I wrote above, the intervals you quote are not directly used as part of the fit, which is done using the full likelihood function, so they will not directly influence the outcome of the fit.
Similarly for your last paragraph, I think I might not have fully understood your question, so please chime in again if I’ve misunderstood what you’re asking. We do expect to regularly see small excesses, even when there is no signal, but it shouldn’t always be the case. The frequency of  excesses of given sizes should be as expected in standard frequentist approaches, e.g. if the background only hypothesis (including the detector modelling, details of the statistical model etc) is a sufficiently good model for whats really happening, we would expect a 1 sigma or larger excess approximately one third of the time. If the statistical model including the background hypothesis isn’t a sufficiently good model of the real underlying process, then the frequency  is unknown as it depends on the way(s) in which the model fails to capture reality.
I hope that helps to clarify.
Best,
Kyle
[1] https://pdg.lbl.gov/2023/web/viewer.html?file=../reviews/rpp2022-rev-statistics.pdf#subsubsection.40.4.2.3 gives details on confidence intervals for poisson counts.
Some more details on the general theory of how these confidence intervals are defined and constructed can be found in section 40.4.2 of the same document: https://pdg.lbl.gov/2023/web/viewer.html?file=../reviews/rpp2022-rev-statistics.pdf#subsection.40.4.2
[2]
using the formula for poisson intervals from [1], implementing the calculation in python, the intervals are given by:
import scipy.stats as st

obs = 43
half_alpha = st.norm(0).cdf(-1)
err_lo = obs - 0.5*st.chi2(2*obs).ppf(half_alpha)
err_hi = 0.5*st.chi2(2*(obs+1)).ppf(1-half_alpha) - obs
print(err_lo, err_hi)
